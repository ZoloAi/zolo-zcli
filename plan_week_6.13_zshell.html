üí° FEATURE CONCEPT:

Problem Statement

  • Current Prompt: Always shows zCLI> regardless of location
  • Issue: Users lose context of current working directory in long sessions
  • Traditional Shells: Show path in prompt (e.g., ~/Projects/zolo-zcli $ )
  • zCLI Difference: We use zPath syntax, not filesystem paths

‚úÖ Implemented Solution

  • Command: where on/off/toggle/status
  • Default (off): zCLI>
  • Enabled (on): zCLI [~.Projects.zolo-zcli]>
  • Session Storage: SESSION_KEY_SHOW_ZPATH_IN_PROMPT
  • Rationale: "where" is a natural question word ("where am I?") and avoids conflicts with zNavigation subsystem

Display Modes (Future Enhancement)

  • where mode simple - Just zCLI>
  • where mode zpath - zCLI [~.Projects.zolo-zcli]>
  • where mode absolute - zCLI [/Users/name/Projects/zolo-zcli]>
  • where mode short - zCLI [zolo-zcli]> (just directory name)
  • where status - Show current mode
ü§î NAMING CONSIDERATIONS:

Option 1: prompt (Recommended ‚≠ê)

  • Pros: Technically accurate, matches shell terminology (PS1)
  • Cons: Might not be immediately obvious to non-technical users
  • Usage: prompt on, prompt off, prompt toggle

Option 2: context

  • Pros: Clear semantic meaning, emphasizes "where am I"
  • Cons: "Context" is broad, might conflict with other features
  • Usage: context show, context hide

Option 3: locate

  • Pros: User-friendly, clear action verb
  • Cons: Sounds like a query command, not a toggle
  • Usage: locate on, locate off

Option 4: breadcrumb

  • Pros: Matches zNavigation's zCrumbs concept
  • Cons: Long word, might confuse with navigation history
  • Usage: breadcrumb on, breadcrumb off

üéØ Recommendation: prompt

Most accurate and aligns with shell conventions. Can be extended to other prompt customizations (colors, timestamps, etc.).
