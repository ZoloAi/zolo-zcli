Current Working Logic (Phase 3 Complete)

üéØ Purpose: zUtils is the plugin management facade for zCLI, responsible for loading plugins from zSpark configuration during boot time and exposing their functions as methods on zcli.utils.

üèóÔ∏è Architecture (Unified Storage): 
zSpark["plugins"] ‚Üí zUtils.load_plugins() ‚Üí zLoader.plugin_cache
                                                            ‚Üì
                                        Single source of truth for all plugins
                                                            ‚Üì
                                    &PluginName.function() ‚Üê zParser
üîÑ Plugin Loading Flow:
  1. Collision Detection: Check if plugin name already exists (Phase 3)
  2. Load Module: Import plugin from file path or module path
  3. Cache in zLoader: Delegate storage to zLoader.plugin_cache
  4. Session Injection: Inject zcli instance into plugin module
  5. Mtime Tracking: Record file modification time for auto-reload (Phase 3)
  6. Secure Exposure: Only expose functions in __all__ (or all public callables with warning)
  7. Stats Update: Increment load counters (Phase 3)
üîí Security (Phase 2):
