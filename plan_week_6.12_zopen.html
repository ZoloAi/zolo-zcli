3-Tier Architecture

Tier 1a: open_paths.py (zPath Resolution) - Foundation
  • Purpose: Translate zPath symbols to filesystem paths
  • Key Methods:
    • resolve_zpath(zpath: str, session: dict) ‚Üí Optional[str] - Main resolver
    • validate_zpath(zpath: str) ‚Üí bool - Validation helper
  • Features:
    • @ (workspace-relative) ‚Üí session["zWorkspace"] + path
    • ~ (absolute) ‚Üí root + path
    • Dot notation parsing (e.g., @.folder.subfolder.file.ext)
    • Error handling for missing workspace context
  • Dependencies: zConfig (session access)
Tier 1b: open_urls.py (URL Handlers) - Foundation
  • Purpose: Handle URL opening in user's preferred browser
  • Key Methods:
    • open_url(url: str, session: dict, display) ‚Üí str - Main URL handler
    • _open_url_browser(url: str, browser: Optional[str], display) ‚Üí str - Browser opener
    • _display_url_fallback(url: str, display) ‚Üí str - Fallback display
  • Features:
    • http/https/www URL detection
    • Browser preference from zMachine.browser
    • Fallback to system default browser
    • URL info display via zDisplay.json_data()
  • Dependencies: zConfig (session), zDisplay (output)
Tier 1c: open_files.py (File Handlers) - Foundation
  • Purpose: Handle local file opening by extension
  • Key Methods:
    • open_file(path: str, session: dict, display, dialog, logger) ‚Üí str - Main file handler
    • _open_html(path: str, display, logger) ‚Üí str - HTML ‚Üí browser
    • _open_text(path: str, session: dict, display, dialog, logger) ‚Üí str - Text ‚Üí IDE
    • _display_file_content(path: str, display, logger) ‚Üí str - Fallback content display
  • Features:
    • Extension-based routing (.html, .txt, .md, .py, .json, .yaml, etc.)
    • IDE preference from zMachine.ide
    • zDialog integration (file creation prompt, IDE selection)
    • Fallback to content display if IDE fails
    • File info display via zDisplay.json_data()
    • Future: PDF, images, archives, media formats
  • Dependencies: zConfig (session), zDisplay (output), zDialog (optional), logger
Tier 1d: open_modules/__init__.py (Package Aggregator)
  • Purpose: Centralized imports and public API exposure
  • Exports:
    • From open_paths: resolve_zpath, validate_zpath
    • From open_urls: open_url
    • From open_files: open_file
  • Pattern: Standard aggregator pattern (like zLoader, zFunc, zDialog)
Tier 2: zOpen.py (Facade)
  • Purpose: Main entry point, routing, and hook execution
  • Key Methods:
    • handle(zHorizontal) - Main handler, routes to appropriate module
    • Type detection: file vs URL vs zPath
    • Hook execution: onSuccess/onFail via zFunc
  • Simplified Logic: Parse ‚Üí Route ‚Üí Execute hook (all heavy lifting in Tier 1 modules)
  • Dependencies: open_modules.*, zFunc (hooks)
Tier 3: __init__.py (Package Root)
  • Exports zOpen class
  • Package-level docstring
  • Usage examples
