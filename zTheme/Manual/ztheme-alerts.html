Alerts

Provide contextual feedback messages for typical user actions with flexible and accessible alert components.

Understanding Alerts

Alerts are attention-grabbing messages that provide important feedback to users. They're perfect for:

  • Success messages: Confirming an action completed successfully
  • Error notifications: Alerting users to problems that need attention
  • Warnings: Cautioning users about potential issues
  • Informational updates: Sharing helpful context or updates
Accessibility: Using color alone to convey meaning isn't enough for screen readers. Always include clear text that explains the alert's purpose. The role="alert" attribute helps screen readers announce important messages immediately.

Basic Alerts

Alerts come in eight contextual variants to match different message types. Use the .zAlert base class with a variant class like .zAlert-success:

A primary alert—check it out!
A secondary alert—check it out!
Your changes have been saved successfully!
Error: Unable to process your request.
Warning: Your session will expire in 5 minutes.
Pro tip: Press Ctrl+S to save your work.
A light alert—great for subtle notices.
A dark alert—high contrast for emphasis. 
<div class="zAlert zAlert-success" role="alert">
  Your changes have been saved successfully!
</div>
<div class="zAlert zAlert-danger" role="alert">
  Error: Unable to process your request.
</div>
<div class="zAlert zAlert-warning" role="alert">
  Warning: Your session will expire in 5 minutes.
</div>
<div class="zAlert zAlert-info" role="alert">
  Pro tip: Press Ctrl+S to save your work.
</div>

Alert Links

Use the .zAlert-link class to style links within alerts. They'll automatically match the alert's color with increased font weight and underline for clarity:

Your profile has been updated! View profile
Payment failed. Please update your payment method.
Your trial ends in 3 days. Upgrade now
New features available! See what's new 
<div class="zAlert zAlert-success" role="alert">
  Your profile has been updated! <a href="#" class="zAlert-link">View profile</a>
</div>

Rich Content Alerts

Alerts can contain more than just text. Add headings, paragraphs, dividers, and other HTML elements for complex messages:

Well done!

You successfully completed the onboarding process. Your account is now fully set up and ready to use.

Need help getting started? Check out our quick start guide.

<div class="zAlert zAlert-success" role="alert">
  <h4 class="zAlert-heading">Well done!</h4>
  <p>You successfully completed the onboarding process...</p>
  <hr>
  <p>Need help? Check out our <a href="#" class="zAlert-link">guide</a>.</p>
</div>
Tip: Use .zAlert-heading on your heading element for proper color inheritance, and spacing utilities like .zmb-0 to fine-tune margins.

Dismissible Alerts

Make alerts dismissible by adding .zAlert-dismissible and a close button. This gives users control over persistent messages:

Notice: Scheduled maintenance tonight at 11 PM EST.
Update: We've improved performance across the platform. 
<div class="zAlert zAlert-warning zAlert-dismissible" role="alert">
  <strong>Notice:</strong> Scheduled maintenance tonight.
  <button type="button" class="zBtn-close" aria-label="Close">×</button>
</div>
JavaScript Integration: The examples above use inline onclick for simplicity. For production, use zTheme.js to handle dismissals with fade-out animations and proper focus management for keyboard users.

When to Use Each Variant

Variant Use Case Example
.zAlert-success Successful actions File uploaded, changes saved
.zAlert-danger Errors and failures Payment failed, invalid input
.zAlert-warning Cautions and potential issues Session expiring, low storage
.zAlert-info Helpful information Tips, updates, tutorials
.zAlert-primary Brand-related messages Welcome messages, highlights
.zAlert-secondary Neutral, less urgent General notices
.zAlert-light Subtle, low priority Hints, suggestions
.zAlert-dark High contrast emphasis Critical announcements

Best Practices

  • Be concise: Keep alert messages brief and actionable
  • Use appropriate variants: Match the alert color to the message severity
  • Include actions: Provide links or buttons for users to resolve issues
  • Don't overuse: Too many alerts can overwhelm users—prioritize what's truly important
  • Screen readers: Always include role="alert" for accessibility
  • Keyboard users: Dismissible alerts should manage focus properly when closed
  • Auto-dismiss carefully: Critical errors shouldn't auto-dismiss—users need time to read them

.zAlert-secondary Secondary neutral alert (gray)
.zAlert-success Success confirmation (green)
.zAlert-danger Error or destructive action (red)
.zAlert-warning Warning or caution (orange)
.zAlert-info Informational message (blue)
.zAlert-light Light, subtle alert
.zAlert-dark Dark, high contrast alert

Additional Classes

Class Description
.zAlert-link Styled links within alerts
.zAlert-heading Heading styling within alerts
.zAlert-dismissible Enables close button positioning
.zBtn-close Close button for dismissible alerts

Animation Classes (Optional)

Class Description
.zFade-in Fade-in animation when alert appears
.zFade-out Fade-out animation when alert is dismissed
