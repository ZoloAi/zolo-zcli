Clearfix

A legacy CSS utility that clears floated elements‚Äîrarely needed in modern Flexbox/Grid layouts, but essential for older code.

Understanding Clearfix

Clearfix is a CSS technique from the "float" layout era (before Flexbox and Grid existed). When elements are floated, they're taken out of the normal document flow, causing their parent container to collapse to zero height. Clearfix solves this by forcing the parent to recognize and contain the floated children.

‚ö†Ô∏è Legacy Technique:

Clearfix was essential in the 2000s-2010s when floats were the primary layout method. In modern CSS (2023+), you should use Flexbox or Grid instead. Only use clearfix when maintaining legacy code or dealing with specific float-based layouts (like wrapping text around images).

‚úÖ When to Use Clearfix:

  • Maintaining legacy code with float-based layouts
  • Text wrapping around images (floated images with text flow)
  • Working with third-party code that uses floats
  • Supporting very old browsers (pre-IE11)

üí° Modern Alternatives:

  • Flexbox: display: flex; - For one-dimensional layouts (rows or columns)
  • Grid: display: grid; - For two-dimensional layouts (rows AND columns)
  • Flow Layout: Normal document flow without floats

These modern techniques handle layout automatically without needing clearfix!

Usage

Simply add the .zClearfix class to the parent element that contains floated children:

<div class="zClearfix">
  <button class="zBtn zFloat-start">Left</button>
  <button class="zBtn zFloat-end">Right</button>
</div>

üí° Remember: Clearfix goes on the parent, not on the floated elements themselves!

How It Works

Clearfix uses a CSS pseudo-element to create an invisible block that clears both floats:

.zClearfix::after {
  content: "";          /* Creates an invisible element */
  display: table;        /* Block formatting context */
  clear: both;           /* Clears left AND right floats */
}

This creates an invisible element after the parent's content that forces the parent to expand and contain the floated children.

Best Practices

‚úÖ DO:

  • Use Flexbox or Grid for new layouts
  • Apply clearfix to the parent element, not the floated children
  • Use floats for text wrapping around images (legitimate use case)
  • Use clearfix when maintaining legacy codebases
  • Combine with float utilities (.zFloat-start, .zFloat-end)

‚ùå DON'T:

  • Don't use floats for page layout - Use Flexbox or Grid instead
  • Don't apply clearfix to the floated elements themselves
  • Don't use clearfix if you're not using floats (it does nothing)
  • Don't build entire layouts with floats in 2023+ (outdated technique)
  • Don't forget about modern CSS alternatives

üí° Migration Path:

If you're maintaining code with lots of floats and clearfixes, consider refactoring to:

  • Flexbox: For horizontal/vertical component layouts
  • Grid: For complex page layouts
  • Modern positioning: For overlays and special cases
