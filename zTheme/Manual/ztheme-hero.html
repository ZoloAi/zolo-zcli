Hero Sections

Full-viewport hero sections for landing pages, welcome screens, and call-to-action areas. Provides centered content with flexible height and color options.

Understanding Hero Sections

A hero section is a prominent introductory area at the top of a page, typically featuring:

  • Large Headings: Eye-catching titles that communicate your main message
  • Supporting Text: Taglines or descriptions that elaborate on the heading
  • Call-to-Action: Buttons or links encouraging user engagement
  • Visual Impact: Background colors, gradients, or images
  • Centered Layout: Content centered both horizontally and vertically
Note: Hero sections work best as the first major content on a page. They should be visually distinct and immediately convey your page's purpose.

Basic Hero

The .zHero class provides a centered, flexible section with default 50vh height:

Welcome to zTheme

A modern CSS framework for beautiful web applications

<section class="zHero">
  <h1>Welcome to zTheme</h1>
  <p>A modern CSS framework for beautiful web applications</p>
</section>

Height Variants

Control the hero section height with these modifier classes:

Full Viewport (.zHero-full)

Fills the viewport (uses 100vh / 100svh / 100dvh) and subtracts any fixed header offset using --zHero-offset so the content always stays centered:

Full Height Hero

This hero fills the entire viewport

<section class="zHero zHero-full zHero-primary">
  <div class="zHero-content">
    <h1>Full Height Hero</h1>
    <p>This hero fills the entire viewport</p>
    <button class="zBtn zBtn-light">Get Started</button>
  </div>
</section>

Tip: if you have a persistent header (like .zNavbar), set --zHero-offset to that height so the hero fills the remaining space:

<section class="zHero zHero-full" style="--zHero-offset: var(--zNavbar-height, 0px);">
  ...
</section>

Available Height Classes

Class Height Use Case
.zHero-full Viewport height − --zHero-offset Landing pages, primary call-to-action, layouts under fixed headers
.zHero-three-quarter 75vh Prominent but not dominant
.zHero-half 50vh (default) Section headers, mid-page heroes
.zHero-quarter 25vh Compact headers, sub-pages

Content Container

Use .zHero-content to constrain content width for better readability:

Constrained Content

This content is limited to 800px width for optimal readability, even on very wide screens. Long paragraphs won't stretch across the entire viewport.

<section class="zHero zHero-half zHero-gradient">
  <div class="zHero-content">
    <h1>Constrained Content</h1>
    <p>This content is limited to 800px width...</p>
  </div>
</section>

Background Images

Add background images with .zHero-cover and optional .zHero-overlay for text readability:

Hero with Background Image

The overlay ensures text remains readable over any image

<section class="zHero zHero-half zHero-cover zHero-overlay" 
         style="background-image: url('path/to/image.jpg');">
  <div class="zHero-content">
    <h1>Hero with Background Image</h1>
    <p>The overlay ensures text remains readable</p>
    <button class="zBtn zBtn-primary">Learn More</button>
  </div>
</section>
Tip: Use .zHero-overlay to add a semi-transparent dark layer over background images, ensuring text remains readable regardless of image content.

Best Practices

  • Keep it focused: One clear message and call-to-action per hero
  • Use hierarchy: Make your main heading significantly larger than supporting text
  • Ensure readability: Always test text visibility over background images/colors
  • Mobile-first: Test on mobile devices - heroes can be overwhelming on small screens
  • Performance: Optimize background images for web (compress, appropriate resolution)
  • Accessibility: Ensure sufficient color contrast (WCAG AA minimum)
  • Loading speed: Hero sections are first impression - optimize for fast load

Usage in zCLI

When using zTheme with zCLI's declarative UI system, you can apply hero styles using the _zClass metadata:

# zUI YAML Definition
zVaF:
  _zClass: "zHero zHero-full zHero-gradient"
  Hero_Section:
    - zDisplay:
        event: header
        content: "Welcome to Our App"
        color: PRIMARY
    - zDisplay:
        event: text
        content: "The best solution for your needs"
        indent: 1

This will automatically wrap the Hero_Section content in a full-height gradient hero section.
