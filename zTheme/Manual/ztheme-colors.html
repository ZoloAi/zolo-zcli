Understanding Color Utilities

Color utilities apply semantic colors to text to convey meaning, establish hierarchy, or match your brand. They work on any element containing text, including headings, paragraphs, links, and spans.

Best Practices

‚úÖ DO

  • Use semantic colors to convey meaning (success = green, danger = red)
  • Use .zText-muted for secondary/supporting text
  • Pair colored text with icons or additional text for accessibility
  • Use .zText-white on dark backgrounds for contrast
  • Use opacity variants (.zText-black-50) for subtle hierarchy

‚ùå DON'T

  • Don't rely on color alone to convey information (accessibility issue)
  • Don't use .zText-light on light backgrounds (poor contrast)
  • Don't overuse colors (creates visual noise and dilutes meaning)
  • Don't forget to test color combinations for WCAG compliance
  • Don't use .zText-warning on white backgrounds without considering readability

Accessibility Considerations

Color Alone is Not Enough

Screen readers cannot detect color changes. Always provide additional context:

‚ùå Bad:

<p class="zText-danger">Failed</p>

‚úÖ Good:

<p class="zText-danger">‚úï <span class="zVisually-hidden">Error: </span>Failed</p>

Color Contrast Requirements

WCAG 2.1 requires minimum contrast ratios:

  • Normal text: 4.5:1 contrast ratio
  • Large text (18px+): 3:1 contrast ratio
  • UI components: 3:1 contrast ratio

üîç Testing Tools: Use browser DevTools Accessibility panel or WebAIM Contrast Checker to verify your color combinations meet WCAG standards.
