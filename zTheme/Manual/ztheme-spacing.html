Spacing Utilities

Control element spacing with margin, padding, and gap utilities.

Understanding Spacing

Spacing utilities control the space around (margin) and inside (padding) elements. Gap utilities control spacing between grid or flex children. These are the most commonly used utilities for creating consistent, responsive layouts.

Margin

Space outside the element. Pushes other elements away.

Padding

Space inside the element. Increases element size.

Gap

Space between grid/flex children. Modern & clean.

Best Practices

  • Consistent scale: Stick to the spacing scale (0-5) for visual consistency
  • Prefer gap: Use .zGap-* for grid/flex layouts instead of margins
  • Vertical rhythm: Use .zmy-* between sections for consistent spacing
  • Component padding: Use .zp-3 or .zp-4 for card/panel internal padding
  • Center blocks: Use .zmx-auto with fixed width for horizontal centering
  • Negative margins sparingly: Only use for intentional overlaps or pull effects
  • Mobile-first: Start with smaller spacing, increase on larger screens if needed
  • RTL support: Use .zms-*/.zme-* instead of .zml-*/.zmr-*

‚ö†Ô∏è Common Mistake: Don't mix margins and gap! If using .zGap-* on a grid/flex container, don't add margins to the children - gap handles spacing automatically!

Accessibility

  • Touch targets: Ensure adequate padding for interactive elements (min 44x44px)
  • Reading flow: Use consistent vertical spacing to guide eye movement
  • Focus visible: Don't remove margin/padding that makes focus indicators visible
  • Content density: Balance whitespace with information density for readability
  • Zoom support: Spacing scales with text zoom (using rem units)

CSS Reference

Spacing Scale

Size Value (rem) Value (px) When to Use
0 0 0px Remove spacing
1 0.25rem 4px Tight spacing, inline elements
2 0.5rem 8px Small spacing, button groups
3 1rem 16px Base spacing - paragraphs, list items
4 1.5rem 24px Card padding, section spacing
5 3rem 48px Large sections, page divisions

Direction Class CSS Property
