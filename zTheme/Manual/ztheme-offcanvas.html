Offcanvas

Slide-out sidebar panels that appear from the edges of the viewport for navigation menus, shopping carts, filters, and contextual content.

Understanding Offcanvas

Offcanvas is a sidebar component that slides in from the screen edges. It's perfect for:

  • Mobile Navigation: Hide main menu off-screen until needed
  • Shopping Carts: Slide-in cart from right without leaving page
  • Filters: Product filters that don't clutter the main view
  • User Profiles: Quick access panels for settings or account info
  • Contextual Content: Help text, notifications, or additional details
Offcanvas vs Modal: Offcanvas slides in from screen edges and is great for navigation/persistent UI. Modals appear in the center and demand immediate attention. Both share similar backdrop and overlay behavior.

Backdrop & Scrolling Options

Control backdrop visibility and body scrolling with data attributes:

Scrolling Allowed

Try scrolling the main page behind this offcanvas. Body scrolling is enabled with data-bs-scroll="true".

The backdrop is visible but doesn't block scrolling.

No Backdrop

No backdrop is shown with data-bs-backdrop="false".

You can interact with the main page while this is open. Clicking outside won't close it.

Both Options

This combines both options:

  • data-bs-scroll="true" - Body scrolling enabled
  • data-bs-backdrop="true" - Backdrop visible

Perfect when you want visual separation but still allow page interaction.

<!-- Allow body scrolling -->
<div class="zOffcanvas zOffcanvas-start" 
     data-bs-scroll="true">
  ...
</div>

<!-- No backdrop -->
<div class="zOffcanvas zOffcanvas-start" 
     data-bs-backdrop="false">
  ...
</div>

<!-- Both options -->

     data-bs-scroll="true" 
     data-bs-backdrop="true">
  ...
</div>

Best Practices

  • Consistent placement: Use the same edge (left/right) for navigation throughout your site
  • Clear triggers: Make buttons obvious with icons (‚ò∞ hamburger, üõí cart)
  • Close options: Always include close button AND backdrop click-to-close
  • Mobile-first: Offcanvas shines on mobile - test thoroughly on small screens
  • Limit content: Keep offcanvas focused - don't overwhelm with too many options
  • Scrollable content: For long content, the body scrolls automatically
  • Right placement: Left for navigation, right for actions (cart, profile)
  • Use sparingly: Too many offcanvas panels can confuse users
  • Keyboard access: Ensure focus management and ESC key closes

Keyboard Navigation

Offcanvas automatically supports keyboard interaction:

  • Tab - Navigate through focusable elements
  • Shift + Tab - Navigate backwards
  • Esc - Close the offcanvas
  • Enter or Space - Activate buttons/links

Focus Management

  • When opened, focus moves to the offcanvas container
  • Focus is trapped inside the offcanvas while open
  • When closed, focus returns to the trigger button
  • Screen readers announce state changes

Data Attributes

Attribute Description

data-bs-target="#id" Target offcanvas ID to show/hide
