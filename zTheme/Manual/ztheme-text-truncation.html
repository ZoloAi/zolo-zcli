Understanding Text Truncation

Text truncation cuts off long text that would overflow its container, replacing the overflow with an ellipsis (‚Ä¶). This keeps layouts clean and prevents text from breaking designs, especially in responsive layouts or when dealing with user-generated content of unknown length.

Perfect for:

  • Long filenames - "document_final_v3‚Ä¶pdf"
  • Email addresses - "verylongemail@exa‚Ä¶"
  • Card titles - Keep card heights consistent
  • Table cells - Prevent horizontal overflow
  • Breadcrumbs - Truncate long path segments
  • User-generated content - Unknown length text

‚ö†Ô∏è Requirements:

  • Element must have display: block or display: inline-block
  • Element must have a defined width or max-width (this is critical!)
  • Won't work on display: inline elements
  • For table cells: Apply max-width directly to the truncate div, not just the column width

Best Practices

‚úÖ DO:

  • Set explicit max-width on truncated inline elements
  • Use tooltips to show full text on hover
  • Provide alternative ways to see full content (expand button, modal)
  • Test with various content lengths
  • Use for non-critical text (filenames, emails, long titles)
  • Consider responsive truncation (different widths per breakpoint)

‚ùå DON'T:

  • Don't truncate critical information that users must read fully
  • Don't use on display: inline elements (won't work!)
  • Don't truncate without providing a way to see full text
  • Don't use for headings that describe page content
  • Don't forget to test with actual user data
  • Don't truncate navigation links (confusing for users)

üí° UX Tip:

Always provide a way to see the full text! Options:

  • Tooltip on hover (use title attribute or .zTooltip)
  • "Show more" expand button
  • Click to modal with full content
  • Link to detail page

Accessibility

‚ö†Ô∏è Important Considerations

  • Screen readers read the full text even if it's visually truncated
  • Truncated content is still in the DOM and accessible
  • Provide visual cues (tooltips, expand buttons) for sighted users
  • Don't truncate critical information without alternative access

Best for Accessibility

  • Use title attribute or tooltips for full text
  • Ensure expand functionality is keyboard accessible
  • Test that screen readers announce full content
  • Don't rely on truncation alone for important info
