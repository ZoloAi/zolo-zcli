Typography

Consistent text styling and inline elements.

Global Settings

zTheme provides consistent typography across all browsers and devices.

  • Native font stack - Uses your device's system font
  • Accessible - Respects user browser settings (default 16px=1rem)
  • Customizable - Override with CSS variables

Default Font Stack:

system-ui, -apple-system, "Segoe UI", Roboto,
"Helvetica Neue", Arial, "Noto Sans",
"Liberation Sans", sans-serif

Paragraphs

The <p> tag is the foundation of text content on the web. It represents a paragraph of text.

This is a regular paragraph. It contains text that flows naturally and wraps to multiple lines when needed.

Paragraphs automatically have spacing between them to separate different thoughts or sections of content.

<p>This is a regular paragraph.</p>
<p>Another paragraph here.</p>

Lead

Make a paragraph stand out by adding .zLead.

This is a lead paragraph. It stands out from regular paragraphs. 

<p class="zLead">
  This is a lead paragraph. It stands out from regular paragraphs.
</p>

Inline Elements

Just like colors need extra context for accessibility, HTML elements work the same way. HTML provides two types of inline elements: semantic (with meaning) and non-semantic (visual only).

Why does this matter?

  • Semantic - Tells screen readers and search engines what the text means, not just how it looks (accessibility & SEO ready)
  • Non-semantic - Visual styling only, like using color alone - no special meaning conveyed

Semantic Elements

These are inline elements - they're used inside paragraphs to style parts of your text. They have meaning for accessibility and SEO:

Highlight

You can use the mark tag to highlight text.

<p>You can use the mark tag to <mark>highlight</mark> text.</p>

Deleted

This text has been deleted from the document.

<p><del>This text has been deleted from the document.</del></p>

Inserted

This text has been added to the document.

<p><ins>This text has been added to the document.</ins></p>

Strong Importance

This text is important.

<p><strong>This text is important.</strong></p>

Emphasis

I really need to finish this.

<p>I <em>really</em> need to finish this.</p>

Small / Fine Print

This is fine print.

<p><small>This is fine print.</small></p>

Non-Semantic Elements

These are also inline elements used inside paragraphs, but for visual styling only (no special meaning):

Bold

This text is bold.

<p><b>This text is bold.</b></p>

Italic

This text is italic.

<p><i>This text is italic.</i></p>

Strikethrough

This text is crossed out.

<p><s>This text is crossed out.</s></p>

Underline

This text is underlined.

<p><u>This text is underlined.</u></p>

zTheme also provides CSS classes as alternatives to HTML elements. Use .zMark instead of <mark>, for example.

.zMark

You can use the mark class to highlight text.

<p>You can use the <span class="zMark">mark class</span> to highlight text.</p>

.zSmall

This is regular text and this is smaller text using the class.

<p>This is regular text and <span class="zSmall">this is smaller text</span> using the class.</p>

.zMuted

This is regular text and this is muted text for less emphasis.

<p>This is regular text and <span class="zMuted">this is muted text</span> for less emphasis.</p>

Headings

Headings organize your content and help both people and search engines understand your page structure.

All six heading levels:

h1. heading

h2. heading

h3. heading

h4. heading

h5. heading
h6. heading

Hierarchy: Use <h1> for main title, <h2> for sections, <h3> for subsections, etc.

<h1>Main Page Title</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Smaller Heading</h5>
<h6>Smallest Heading</h6>

Heading Utility Classes

Sometimes you need text that looks like a heading without affecting your page structure. Use .zH1 through .zH6 classes instead of <h1> through <h6> for visual styling without semantic meaning.

My Blog Section

Introduction

Some text here...

A big styled quote that looks important

More text continues...

<h4>My Blog Section</h4>
<h5>Introduction</h5>
<p>Some text here...</p>
<p class="zH4">A big styled quote that looks important</p>
<p>More text continues...</p>

In this example, the quote looks like an <h4>, but doesn't break the heading hierarchy or confuse search engines.

Abbreviations

The <abbr> tag marks abbreviations and acronyms. When you hover over them, a tooltip appears with the full meaning.

Try it: Hover your mouse over the underlined words below and keep it still for about 1 second - a tooltip will appear with the full meaning.

Our API supports JSON and XML.

Contact Dr. Smith at NASA headquarters.

Please reply ASAP regarding the HTML update.

<p>Our <abbr title="Application Programming Interface">API</abbr> supports JSON.</p>
<p>Contact <abbr title="Doctor">Dr.</abbr> Smith at NASA.</p>
<p>Reply <abbr title="As Soon As Possible" class="zAcronym">ASAP</abbr>.</p>

.zAcronym makes all-caps acronyms (NASA, HTML, API) slightly smaller so they don't overpower surrounding text.

Blockquotes

The <blockquote> element is a semantic HTML tag for quoting content from another source. It tells screen readers and search engines "this is a quote" - important for SEO and accessibility. zTheme automatically styles all blockquotes with indentation, a left border, and larger font.

A well-known quote, contained in a blockquote element.

<blockquote>
  <p>A well-known quote, contained in a blockquote element.</p>
</blockquote>

Blockquote Utility Class

Want to style a <div> or other element to look like a blockquote without the semantic meaning? Use the .zBlockquote utility class:

This looks like a blockquote but uses a div element.

<div class="zBlockquote">
  <p>This looks like a blockquote but uses a div element.</p>
</div>

Naming a Source

To properly attribute a quote, use semantic HTML elements: <figure> wraps the entire quote with source, <figcaption> contains the attribution, and <cite> marks the source title. Use the .zBlockquote-footer utility class to style the caption.

<figure>
  <blockquote>
    <p>A well-known quote, contained in a blockquote element.</p>
  </blockquote>
  <figcaption class="zBlockquote-footer">
    Someone famous in <cite title="Source Title">Source Title</cite>
  </figcaption>
</figure>

Semantic elements: <figure>, <blockquote>, <figcaption>, and <cite> are all semantic HTML - they tell screen readers and search engines the structure and meaning. Only .zBlockquote-footer is a zTheme utility class for visual styling.

Lists

Lists are semantic HTML elements that group related items. Use <ul> for unordered (bullet) lists, <ol> for ordered (numbered) lists, and <dl> for description lists. All use <li> for list items.

Unordered Lists

Use <ul> when the order of items doesn't matter. Items are marked with bullets by default.

  • Coffee
  • Tea
  • Milk
<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>

Ordered Lists

Use <ol> when the sequence matters. Items are numbered automatically.

  1. First step
  2. Second step
  3. Third step
<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>

Description Lists

Use <dl> for term-definition pairs. <dt> is the term, <dd> is the description.

HTML
Hypertext Markup Language
CSS
Cascading Style Sheets
<dl>
  <dt>HTML</dt>
  <dd>Hypertext Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

Unstyled Lists

Need to remove bullets and padding? Use the .zList-unstyled utility class. This only affects immediate children - nested lists remain styled.

  • No bullets here
  • Clean and simple
  • Still a semantic list
  • Nested lists:
    • Keep their bullets
    • And normal styling
<ul class="zList-unstyled">
  <li>No bullets here</li>
  <li>Clean and simple</li>
  <li>Still a semantic list</li>
</ul>

Inline Lists

Display list items horizontally (like navigation) with .zList-inline on the list and .zList-inline-item on each item.

  • Home
  • About
  • Contact
<ul class="zList-inline">
  <li class="zList-inline-item">Home</li>
  <li class="zList-inline-item">About</li>
  <li class="zList-inline-item">Contact</li>
</ul>

Tip: Inline lists are perfect for navigation menus - search engines understand the semantic structure while users see a horizontal layout.

Responsive Font Sizes

All typography in zTheme automatically adjusts to look great on any device - from phones to desktop monitors. Text scales proportionally, maintaining readability without manual adjustments.

We use rem units for all font sizes. This means if a user increases their browser's default font size (for better readability), your text scales up accordingly - much better for accessibility!
