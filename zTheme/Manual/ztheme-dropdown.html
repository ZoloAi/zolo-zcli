Dropdowns

Toggle contextual overlays for displaying lists of links, actions, and more with click-based dropdown menus.

Understanding Dropdowns

Dropdowns are contextual menus that appear when users click a button or link. They're perfect for:

  • Actions: Show a list of actions users can take
  • Navigation: Display additional menu items
  • Filters: Provide sorting or filtering options
  • Settings: Show configuration choices
  • User menus: Display profile, logout, settings
Accessibility: Dropdowns support keyboard navigation (↑ ↓ arrow keys to navigate, Escape to close) and use aria-expanded to communicate state to screen readers.

Basic Dropdown

Wrap the toggle button and menu in a .zDropdown container:

  • Action
  • Another action
  • Something else here
<div class="zDropdown">
  <button class="zBtn zBtn-secondary zDropdown-toggle" 
          type="button" 
          data-bs-toggle="dropdown" 
          aria-expanded="false">
    Dropdown Button
  </button>
  <ul class="zDropdown-menu">
    <li><a class="zDropdown-item" href="#">Action</a></li>
    <li><a class="zDropdown-item" href="#">Another action</a></li>
    <li><a class="zDropdown-item" href="#">Something else here</a></li>
  </ul>
</div>

How it works:

  • .zDropdown — Container with position: relative
  • .zDropdown-toggle — Button/link that opens the menu
  • data-bs-toggle="dropdown" — Tells zTheme to attach click handler
  • .zDropdown-menu — The hidden menu that appears on click
  • .zDropdown-item — Individual menu items (links or buttons)

Best Practices

  • Keep menus focused: Limit to 5-7 items for easy scanning
  • Use semantic HTML: Use <button> for actions, <a> for navigation
  • Group related items: Use headers and dividers to organize content
  • Show current state: Use .zActive for the current selection
  • Disable unavailable options: Use .zDisabled instead of hiding
  • Mobile-friendly: Dropdowns work on touch devices—test on mobile!
  • Keyboard accessible: Use arrow keys to navigate, Escape to close
  • Clear labels: Button text should indicate what the menu contains
Avoid dropdown overload: If you have more than 10 items, consider using a search field, tabbed interface, or navigation page instead.

CSS Reference

Dropdown Classes

Class Description
.zDropdown Container for dropdown (default direction: down)
.zDropup Container for dropup (direction: up)
.zDropend Container for dropend (direction: right)
.zDropstart Container for dropstart (direction: left)
.zDropdown-toggle Button/link that triggers the menu
.zDropdown-toggle-split Split button dropdown toggle styling
.zDropdown-menu The dropdown menu container
.zShow Shows the menu (managed by JavaScript)

Menu Item Classes

Class Description
.zDropdown-item Individual menu item (link or button)
.zActive Highlighted/current menu item
.zDisabled Non-clickable/disabled menu item
.zDropdown-header Section header within menu
.zDropdown-divider Horizontal divider (<hr>)
.zDropdown-item-text Non-interactive text in menu

Variants

Class Description
.zDropdown-menu-dark Dark themed menu
.zDropdown-menu-end Right-aligned menu
.zDropdown-menu-start Left-aligned menu (default)

Data Attributes

Attribute Required Description
data-bs-toggle="dropdown" Yes Identifies element as a dropdown trigger
aria-expanded Recommended Accessibility: current state (auto-managed)
Auto-Initialization: Just add the data attributes and zTheme.js handles the rest. For dynamically added dropdowns, call window.zTheme.initDropdown().
