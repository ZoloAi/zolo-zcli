Understanding Display Utilities

Display utilities control how elements are rendered using the CSS display property. They're essential for responsive design, allowing you to show/hide elements at different screen sizes and control layout behavior.

üí° Pro Tip: Combine display utilities with breakpoint suffixes (e.g., .zD-none .zD-md-block) to hide elements on mobile but show them on desktop!

Class Naming

Display utility classes follow this pattern:

  • .zD-{value} - Applies at all screen sizes (xs and up)
  • .zD-{breakpoint}-{value} - Applies at specified breakpoint and up
  • .zD-print-{value} - Applies only when printing

Available values:

  • none - display: none
  • inline - display: inline
  • inline-block - display: inline-block
  • block - display: block
  • grid - display: grid
  • table - display: table
  • table-row - display: table-row
  • table-cell - display: table-cell
  • flex - display: flex
  • inline-flex - display: inline-flex

Best Practices

‚úÖ DO

  • Use .zD-none for mobile-first design (hide by default, show on larger screens)
  • Combine display utilities for responsive behavior (.zD-none .zD-md-block)
  • Use .zD-flex and .zD-grid for modern, flexible layouts
  • Hide non-essential content when printing (.zD-print-none on navigation, ads)
  • Test responsive behavior by resizing your browser window

‚ùå DON'T

  • Don't create separate mobile and desktop versions (use responsive utilities instead)
  • Don't forget to test print view if you're using print utilities
  • Don't overuse .zD-none - hiding content from screen readers is a separate concern
  • Don't use display utilities for accessibility hiding (use .zVisually-hidden)
  • Don't forget that .zD-none completely removes elements from layout (use visibility if you need to preserve space)
