Sizing Utilities

Control element dimensions with width, height, and viewport sizing utilities.

Understanding Sizing

Sizing utilities let you control element dimensions relative to their parent container (percentage-based) or relative to the viewport (vw/vh units). Use these utilities for responsive layouts without custom CSS.

üí° Key Concept: Percentage widths are relative to the parent element, while viewport units (vw/vh) are relative to the browser window. Choose based on whether you need container-based or window-based sizing!

Best Practices

  • Responsive images: Always use .zMw-100 on images to prevent overflow
  • Full-width sections: Use .zW-100 for container-based or .zVw-100 for viewport-based
  • Hero sections: .zVh-100 for full-screen, but test on mobile (address bar issues)
  • Sidebars: Use .zW-25/.zW-33 for sidebars, .zW-75/.zW-66 for content
  • Mobile-first: Use viewport units cautiously - they can cause horizontal scroll
  • Max constraints: Combine .zW-100 with inline max-width for responsive yet constrained elements
  • Height requires context: Percentage heights only work if parent has defined height
  • Viewport units for breakout: vw can break out of containers for full-width effects

‚ö†Ô∏è Common Pitfall: Percentage heights (.zH-50) don't work unless the parent has an explicit height! Either give the parent a fixed height, vh height, or use min-height instead.

Accessibility

  • Zoom considerations: Fixed viewport sizes may cause issues at high zoom levels
  • Overflow: Always test that content doesn't get cut off at different screen sizes
  • Touch targets: Ensure interactive elements maintain sufficient size (min 44x44px)
  • Text readability: Don't let width constraints make text too narrow (min ~45 characters)
  • Landscape/portrait: Test vh units in both orientations on mobile
