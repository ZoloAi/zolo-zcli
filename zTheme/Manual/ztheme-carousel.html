Carousel

A slideshow component for cycling through images or content slides with CSS 3D transforms and JavaScript.

✨ Auto-Initialization

Carousels on this page are automatically initialized by zTheme.js. The demos below cycle every 5 seconds—hover over them to pause!

⚠️ Accessibility Notice

Carousels are not automatically accessible. Ensure images have alt attributes, controls are keyboard accessible, and users can pause auto-cycling (especially important for motion sensitivity).

Understanding Carousels

Carousels (also called sliders or slideshows) cycle through a series of content—typically images, but can be text, cards, or custom markup. They're useful for:

  • Image galleries: Showcasing product photos, portfolio work, or featured images
  • Hero sections: Rotating banners on homepages or landing pages
  • Testimonials: Cycling through customer reviews or quotes
  • Feature highlights: Presenting multiple product features or benefits
Accessibility Concerns: Carousels are not automatically accessible. Users with screen readers or keyboard-only navigation may struggle. Use carousels sparingly and ensure:
  • Previous/next controls are keyboard accessible
  • Users can pause auto-cycling (especially important for motion sensitivity)
  • Images have proper alt attributes
  • Consider if a simple image grid might be better for your use case

The Semantic Foundation

At its core, a carousel is just a container with multiple content items. Without styling or JavaScript, all slides are visible as a simple stack:

Slide 1
Slide 2
Slide 3 
<div>
  <div>
    <div>Slide 1</div>
    <div>Slide 2</div>
    <div>Slide 3</div>
  </div>
</div>
Non-Semantic HTML: Carousels use generic <div> containers because they're purely presentational. The semantic meaning comes from the content inside each slide (images, headings, paragraphs, etc.).

Enhanced Carousel with .zCarousel

The .zCarousel classes transform the stack into a slideshow with transitions. Only one slide is visible at a time, with smooth animations between slides:

The transformation: .zCarousel hides all slides except the one marked .zActive, with smooth CSS transitions. Add data-ride="true" to auto-start cycling.

zTheme Pattern: The <div> structure provides the container (unstyled stack). The .zCarousel classes add the slideshow styling and transitions. JavaScript adds interactivity.

Image Carousels

The most common use case—cycling through images. Use responsive image sizing for mobile-friendly slideshows:

<div class="zCarousel">
  <div class="zCarousel-inner">
    <div class="zCarousel-item zActive">
      <img src="image1.jpg" style="width: 100%; display: block;" alt="...">
    </div>
    <div class="zCarousel-item">
      <img src="image2.jpg" style="width: 100%; display: block;" alt="...">
    </div>
  </div>
</div>

Note: Use width: 100%; display: block; on images to prevent layout shifts and ensure they fill the carousel width.

Interactive Features

Carousels support multiple interaction methods automatically via zTheme.js:

Navigation Methods

  • Previous/Next Buttons: Click arrows to navigate manually
  • Indicators (Dots): Click dots to jump to specific slides
  • Keyboard Navigation: Use ← → arrow keys (or ↑ ↓ for vertical)
  • Touch Swipe: Swipe left/right on mobile (or up/down for vertical)
  • Auto-Cycling: Slides change automatically with data-ride="true"
  • Pause on Hover: Hover over carousel to pause auto-cycling
Try it now! The carousels on this page demonstrate all these features. Try clicking controls, swiping on mobile, or using arrow keys!

User Customization & Overrides

zTheme is designed to be customized. You can override any default styling or behavior by loading your custom CSS/JS after zTheme files:

CSS Customization

Override carousel styling by redefining classes in your custom stylesheet:

<!-- In your HTML -->
<link rel="stylesheet" href="/static/css/zTheme.css">
<link rel="stylesheet" href="/static/css/my-custom.css">  <!-- Your overrides -->

<!-- In my-custom.css -->
/* Custom carousel control colors */
.zCarousel-control-prev,
.zCarousel-control-next {
  opacity: 0.8;
  background: rgba(0, 0, 0, 0.3);
}

/* Custom progress bar color */
.zCarousel-progress-bar {
  background-color: #ff6b6b;
}

/* Custom transition duration */
.zCarousel-item {
  transition: transform 1s ease-in-out;
}

JavaScript Hooks (Swiper-Inspired)

Add custom behavior using data attributes and global callback functions:

<!-- HTML: Define hooks via data attributes -->
<div class="zCarousel" 
     data-ride="true"
     data-on-slide-change="mySlideHandler"
     data-on-transition-start="myStartHandler"
     data-on-transition-end="myEndHandler">
  <div class="zCarousel-inner">...</div>
</div>

<!-- JavaScript: Define callback functions -->
<script>
  // Called whenever slide changes
  function mySlideHandler(event) {
    console.log('Slide changed to:', event.index);
    console.log('Direction:', event.direction);
  }
  
  // Called when transition starts
  function myStartHandler(event) {
    console.log('Transitioning from', event.from, 'to', event.to);
  }
  
  // Called when transition completes
  function myEndHandler(event) {
    console.log('Transition complete. Current slide:', event.index);
    // Example: Update external UI, trigger analytics, etc.
  }
</script>

CSS Variables for Easy Theming

Customize carousel appearance globally using CSS variables:

:root {
  /* Override carousel colors */
  --carousel-control-opacity: 0.7;
  --carousel-control-hover-opacity: 1;
  --carousel-indicator-active-bg: var(--color-primary);
  --carousel-progress-bg: rgba(255, 255, 255, 0.5);
  --carousel-progress-bar-bg: var(--color-success);
  --carousel-caption-color: var(--color-white);
  --carousel-transition-duration: 0.6s;
}
Load Order Matters: Always load your custom CSS after zTheme CSS files. This ensures your overrides take precedence due to CSS cascade rules. Same principle applies to JavaScript—define your hooks before zTheme initializes.

Choosing the Right Transition

Transition Best For Avoid When
Horizontal Slide (default) Image galleries, product showcases Text-heavy content (distracting)
Fade Testimonials, quotes, text content Image galleries (less dynamic)
Vertical Slide News feeds, social media posts Wide horizontal content

Best Practices

  • Use sparingly: Carousels hide content—users often miss slides beyond the first
  • Keep it simple: 3-5 slides maximum; more becomes overwhelming
  • Choose appropriate transition: Fade for text, slide for images, vertical for feeds
  • Provide controls: Always include previous/next buttons for user control
  • Add indicators: Dots help users understand total slide count and position
  • Pause on hover: Stop auto-cycling when users hover (built-in accessibility)
  • Accessible alt text: Every image needs a descriptive alt attribute
  • Touch-friendly: Touch swipe is enabled by default on mobile
  • Keyboard accessible: Arrow keys work by default
  • Motion sensitivity: Slide transitions respect prefers-reduced-motion
  • Performance: Optimize images for fast loading
  • Consider alternatives: Sometimes a grid or tabbed interface is more usable
When NOT to Use Carousels:
  • Critical information users must see (they'll miss it)
  • More than 5 items (too many slides to navigate)
  • Content that benefits from comparison (use a grid instead)
  • Pages targeting motion-sensitive users exclusively

CSS Reference

Carousel Classes

Class Description
.zCarousel Base carousel container (horizontal slide, default)
.zCarousel-inner Wrapper for carousel items (handles overflow)
.zCarousel-item Individual slide container
.zActive Marks the currently visible slide
.zCarousel-fade Crossfade transition (opacity)
.zCarousel-vertical Vertical slide transition (zTheme extension)

Data Attributes (JavaScript Configuration)

Attribute Default Description
data-ride false Auto-start cycling on page load
data-interval 5000 Time between slides (milliseconds)
data-wrap true Loop back to first slide after last
data-pause true Pause cycling on hover
data-touch true Enable touch swipe on mobile devices

JavaScript Event Hooks (Swiper-Inspired)

Hook Fires When Event Data
data-on-slide-change Active slide changes { index, direction }
data-on-transition-start Slide transition begins { from, to, direction }
data-on-transition-end Slide transition completes { index, direction }

Example use cases: Analytics tracking, syncing external UI, triggering animations, updating related content.
