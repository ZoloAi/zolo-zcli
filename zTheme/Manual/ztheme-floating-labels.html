Floating Labels

Create modern, space-efficient forms where labels animate into placeholders.

Understanding Floating Labels

Floating labels are a modern UX pattern where the label starts inside the input field (like a placeholder) and floats above when the user focuses or enters text. This creates a clean, minimal interface while maintaining accessibility.

Benefits:

  • Space-efficient: Label and input occupy the same vertical space initially
  • Context retention: Unlike pure placeholders, the label remains visible after input
  • Modern aesthetic: Clean, minimal appearance popular in material design
  • CSS-only: No JavaScript required‚Äîuses :placeholder-shown pseudo-class
Accessibility: Floating labels maintain proper <label> and <input> association via for/id attributes. Screen readers announce labels correctly, and keyboard navigation works as expected. The semantic HTML remains intact‚Äîthe floating behavior is pure visual enhancement.

When to use: Login/signup forms, contact forms, checkout flows, profile editors‚Äîanywhere you want a modern, minimal aesthetic without sacrificing usability.

Basic Floating Labels

Wrap an .zInput and <label> in .zFloating. The input must have a placeholder (required for CSS), and the input must come before the label in HTML (CSS uses sibling selector ~).

<div class="zFloating">
  <input type="email" class="zInput" id="emailFloat" placeholder="name@example.com">
  <label for="emailFloat">Email Address</label>
</div>

<div class="zFloating">
  <input type="password" class="zInput" id="passwordFloat" placeholder="Password">
  <label for="passwordFloat">Password</label>
</div>
Important: The placeholder attribute is required for the CSS to work (:placeholder-shown pseudo-class). The actual placeholder text is hidden via CSS, so it doesn't matter what you write‚Äîbut it must be present!

With Pre-filled Values

When an input has a value attribute, the label automatically floats to its elevated position:

<div class="zFloating">
  <input type="email" class="zInput" id="prefilled" 
         placeholder="name@example.com" value="sarah.chen@company.com">
  <label for="prefilled">Email Address</label>
</div>

With Textareas

Floating labels work with <textarea> elements. Control height with inline styles or CSS (not the rows attribute):

<!-- Default height -->
<div class="zFloating">
  <textarea class="zInput" id="messageDefault" placeholder="Leave a message"></textarea>
  <label for="messageDefault">Message</label>
</div>

<!-- Custom height -->
<div class="zFloating">
  <textarea class="zInput" id="messageCustom" 
            placeholder="Leave a message" style="height: 120px"></textarea>
  <label for="messageCustom">Message</label>
</div>

With Select Dropdowns

Floating labels work with <select> elements. The label always stays in the floated position (since selects always have a value).

<div class="zFloating">
  <select class="zSelect" id="countrySelect">

In Grid Layouts

Floating labels work seamlessly with the zTheme grid system:

<div class="zRow zG-2">
  <div class="zCol-md">
    <div class="zFloating">
      <input type="text" class="zInput" id="firstName" placeholder="First">
      <label for="firstName">First Name</label>
    </div>
  </div>

Best Practices

Always Provide Placeholders

The placeholder attribute is required for the CSS to work:

<!-- Good: Has placeholder -->
<input type="email" class="zInput" placeholder="name@example.com">

<!-- Bad: Missing placeholder (floating won't work) -->
<input type="email" class="zInput">

Input Before Label

HTML order matters‚Äîinput must come before label for the CSS sibling selector (~) to work:

<!-- Good: Input first -->
<div class="zFloating">
  <input type="text" class="zInput" id="name" placeholder="Name">
  <label for="name">Your Name</label>
</div>

<!-- Bad: Label first (won't float) -->
<div class="zFloating">
  <label for="name">Your Name</label>
  <input type="text" class="zInput" id="name" placeholder="Name">
</div>

Use Meaningful Labels

Consider Mobile Users

Floating labels work well on mobile, but ensure touch targets are large enough. The default --form-floating-height (3.5rem) provides good mobile usability.
