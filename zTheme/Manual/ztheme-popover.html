Popover

Display rich content in floating overlays triggered by clicks, hovers, or focus events.

Understanding Popovers

Popovers are contextual overlays that display rich content (text, images, buttons, etc.) when triggered. Unlike tooltips which show simple text, popovers can contain HTML, headers, and multiple paragraphs.

When to use:

  • Additional context - Show supplementary information without cluttering the interface
  • Help text - Provide detailed explanations for complex features
  • User prompts - Guide users through multi-step processes
  • Mini forms - Quick actions like adding tags or comments

‚ö†Ô∏è When NOT to use popovers:

  • Critical information - Don't hide essential content that users must see
  • Mobile primary actions - Popovers can be hard to dismiss on touch devices
  • Too much content - If it doesn't fit in ~250px, use a modal instead
  • Simple tooltips - For single-line hints, use tooltips (lighter, faster)

Basic Popover

Click the button to reveal a popover. The popover appears on click and can be dismissed by clicking outside or the button again.

<button class="zBtn zBtn-primary" 
        data-bs-toggle="popover" 
        data-bs-content="This is a basic popover with simple content.">
  Click me
</button>

Popover with Title

Add a data-bs-title attribute to include a header section.

<button class="zBtn zBtn-success" 
        data-bs-toggle="popover" 
        data-bs-title="Popover Title"
        data-bs-content="Popovers with titles help organize content.">
  Popover with Header
</button>

Placement Options

Control where the popover appears relative to the trigger element using data-bs-placement.

<button data-bs-toggle="popover" data-bs-placement="top">Top</button>
<button data-bs-toggle="popover" data-bs-placement="end">Right</button>
<button data-bs-toggle="popover" data-bs-placement="bottom">Bottom</button>
<button data-bs-toggle="popover" data-bs-placement="start">Left</button>

Trigger Options

Choose how the popover is triggered: click (default), hover, or focus.

Click Trigger (Default)

Popover appears on click and stays open until dismissed.

<button data-bs-toggle="popover" 
        data-bs-trigger="click"
        data-bs-content="...">
  Click Trigger
</button>

Hover Trigger

Popover appears on mouse hover and disappears when the mouse leaves.

<button data-bs-toggle="popover" 
        data-bs-trigger="hover"
        data-bs-content="...">
  Hover Trigger
</button>

Focus Trigger

Popover appears when the element receives focus (useful for form inputs).

<button data-bs-toggle="popover" 
        data-bs-trigger="focus"
        data-bs-content="...">
  Focus Trigger
</button>

Color Variants

Use data-bs-custom-class to add contextual color themes to popovers.

<button data-bs-toggle="popover" 
        data-bs-custom-class="zPopover-primary"
        data-bs-content="...">Primary</button>

<button data-bs-toggle="popover" 
        data-bs-custom-class="zPopover-success"
        data-bs-content="...">Success</button>

<button data-bs-toggle="popover" 
        data-bs-custom-class="zPopover-danger"
        data-bs-content="...">Danger</button>

Best Practices

‚úÖ DO:

  • Keep content concise (under 250 words)
  • Use clear, actionable titles
  • Ensure popovers work on mobile (avoid hover-only triggers)
  • Provide clear ways to dismiss (outside click, ESC key)
  • Use appropriate placement to avoid covering important content
  • Test that popovers stay within viewport bounds

‚ùå DON'T:

  • Hide critical information in popovers
  • Use hover triggers on mobile-first designs
  • Stack multiple popovers on top of each other
  • Make popovers the only way to access features
  • Use popovers for long-form content (use modals instead)
  • Forget to handle keyboard navigation (ESC to close)

Accessibility

Popovers are accessible by default, but follow these guidelines for the best experience:

Built-in Accessibility Features

  • ARIA Role: Popovers have role="tooltip" for screen readers
  • Keyboard Support: ESC key closes popovers (click triggers)
  • Focus Management: Focus returns to trigger when popover closes
  • Outside Click: Clicking outside automatically dismisses

Additional Considerations

  • Descriptive Triggers: Button text should indicate what the popover contains
  • Avoid Hover-Only on Mobile: Touch devices can't "hover" - use click instead
  • Test with Screen Readers: Ensure popover content is announced properly
  • Color Contrast: Maintain WCAG AA contrast ratios (4.5:1 for text)

Data Attributes Reference

All popover options are configured via HTML data attributes:

Attribute Values Default Description
data-bs-toggle popover - Required. Enables popover functionality.
data-bs-content String (HTML allowed) "" Required. The main popover content.
data-bs-title String "" Optional. Popover header text.
data-bs-placement top, bottom, start, end bottom Where the popover appears relative to trigger.
data-bs-trigger click, hover, focus click How the popover is triggered.
data-bs-custom-class CSS class name "" Add custom styling (e.g., zPopover-primary).

.zPopover-end Placement to the right

.zPopover-success Success color variant
.zPopover-danger Danger color variant
.zPopover-warning Warning color variant
.zShow Visible state (added by JS)
