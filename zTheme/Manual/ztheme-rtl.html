Dynamic Language Switching

For applications that support multiple languages, switch direction dynamically with JavaScript: 

// Switch to RTL (Hebrew, Arabic, etc.)
function setRTL() {
  document.documentElement.setAttribute('dir', 'rtl');
  document.documentElement.setAttribute('lang', 'he'); // or 'ar', 'fa', 'ur'
}

// Switch to LTR (English, etc.)
function setLTR() {
  document.documentElement.setAttribute('dir', 'ltr');
  document.documentElement.setAttribute('lang', 'en');
}

// Toggle direction
function toggleDirection() {
  const currentDir = document.documentElement.getAttribute('dir');
  if (currentDir === 'rtl') {
    setLTR();
  } else {
    setRTL();
  }
}

// Auto-detect from user preference or browser
function detectDirection() {
  const userLang = navigator.language || navigator.userLanguage;
  const rtlLanguages = ['he', 'ar', 'fa', 'ur'];
  
  if (rtlLanguages.some(lang => userLang.startsWith(lang))) {
    setRTL();
  } else {
    setLTR();
  }
}

Server-Side (Python/Flask Example)

from flask import render_template, session

@app.route('/')
def index():
    # Get user's language preference
    lang = session.get('language', 'en')
    dir = 'rtl' if lang in ['he', 'ar', 'fa', 'ur'] else 'ltr'
    
    return render_template('index.html', lang=lang, dir=dir)

What Needs Manual RTL Handling

Some elements require manual adjustment for RTL:

1. Directional Icons

Icons with direction (arrows, chevrons) should flip in RTL: 

/* CSS approach */
[dir="rtl"] .icon-arrow-right {
  transform: scaleX(-1); /* Flip horizontally */
}

/* Or use separate icons */
<span class="icon-arrow-right">â†’</span> <!-- LTR -->
<span class="icon-arrow-left" style="display:none;">â†</span> <!-- RTL -->

2. Custom Positioning

If you use custom left/right in inline styles, add RTL variants: 

/* Instead of: */
style="left: 20px;"

/* Use CSS logical properties: */
style="inset-inline-start: 20px;" /* Modern browsers */

/* Or use classes: */
class="zStart-0" /* zTheme utility */

3. Shadows & Gradients

Box shadows and gradients with direction should flip: 

[dir="rtl"] .card {
  box-shadow: -2px 2px 8px rgba(0,0,0,0.1); /* Flip shadow */
}

[dir="rtl"] .gradient {
  background: linear-gradient(to left, blue, red); /* Flip gradient */
}

4. Text in Images

Images with embedded text need separate RTL versions or should be avoided.

Best Practices

  • Use start/end everywhere: Always prefer start/end over left/right
  • Test early: Don't wait until the end to add RTL support
  • Real content: Test with actual Hebrew/Arabic text, not Lorem Ipsum
  • Mobile testing: RTL on mobile can expose unique layout issues
  • Icons with meaning: Flip directional icons (arrows), but not semantic ones (âœ“, âœ—)
  • Forms: Test form validation and error messages in RTL
  • Numbers: Remember numbers stay LTR even in RTL text
  • Mixed content: Handle mixed LTR/RTL content with dir="auto"

Accessibility & RTL

  • lang attribute: Always set lang="he" or lang="ar" with RTL
  • Screen readers: Proper lang helps screen readers pronounce correctly
  • Keyboard navigation: Arrow keys should follow visual direction in RTL
  • ARIA labels: Translate ARIA labels for RTL languages
  • Focus order: Ensure tab order follows RTL visual flow
