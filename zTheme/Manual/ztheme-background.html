Understanding Background Utilities

Background utilities apply color to elements to convey meaning, establish visual hierarchy, or create zones. Unlike color themes for components, background utilities are layout-level helpers that work with any element.

‚ö†Ô∏è Important: Background utilities do NOT set text color. Always pair them with .zText-white or .zText-dark for proper contrast!

Background Colors

All theme colors are available as background utilities:

.zBg-primary - Primary brand color (Intention)
.zBg-secondary - Secondary brand color (Validation)
.zBg-success - Success states (Manifestation)
.zBg-info - Informational content (Perception)
.zBg-warning - Warning states (Caution)
.zBg-danger - Error/danger states (Decisive)
.zBg-light - Light neutral background
.zBg-dark - Dark neutral background
.zBg-white - Pure white background
.zBg-body - Body background color (warm natural tone)
.zBg-transparent - Transparent background

Background Gradients

Add .zBg-gradient to any background color to apply a subtle gradient overlay. The gradient starts with semi-transparent white at the top and fades to transparent at the bottom, creating visual depth.

.zBg-primary.zBg-gradient - Primary with gradient overlay
.zBg-success.zBg-gradient - Success with gradient overlay
.zBg-info.zBg-gradient - Info with gradient overlay
.zBg-danger.zBg-gradient - Danger with gradient overlay
.zBg-dark.zBg-gradient - Dark with gradient overlay

</div>

Best Practices

‚úÖ DO

  • Always pair background utilities with text color utilities for proper contrast
  • Use semantic colors to convey meaning (success = green, danger = red)
  • Test color combinations for accessibility (WCAG 2.1 AA minimum)
  • Use gradients sparingly for visual hierarchy
  • Leverage .zBg-body for sections that should match page background

‚ùå DON'T

  • Don't use background utilities without considering text contrast
  • Don't overuse gradients (they can slow page rendering if excessive)
  • Don't rely solely on color to convey information (use icons/text too)
  • Don't forget to test on different screen brightness levels
  • Don't use .zBg-warning with white text (poor contrast!)

Accessibility Considerations

Color Contrast Requirements

WCAG 2.1 requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text (18px+).
