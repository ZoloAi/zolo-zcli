Reboot

CSS reset for consistent cross-browser styling

What is Reboot?

Different browsers style HTML elements differently by default. Reboot fixes this by giving all basic HTML elements (headings, paragraphs, lists, tables, etc.) consistent styling across all browsers.

Think of it as a "fresh start" for your HTML elements - everything looks the same whether someone views your site in Chrome, Firefox, Safari, or any other browser.

This is a quick reference page showing what Reboot does. For detailed styling options, see the following pages:

  • Typography - Headings, paragraphs, lists, and text styling
  • Images - Image styling and responsive images
  • Tables - Table components and styling

Approach

Before diving into Reboot's principles, let's understand CSS units:

Understanding em and rem

What is em?

An em is relative to the parent element's font size. If a parent has font-size: 20px, then 1em = 20px for its children.

The problem: em compounds. If you nest elements, each level multiplies:

  • Parent: font-size: 20px
  • Child with font-size: 1.5em = 30px (20 × 1.5)
  • Grandchild with font-size: 1.5em = 45px (30 × 1.5) ← Gets bigger unexpectedly!

What is rem?

rem = "root em" - always relative to the root <html> element's font size (usually 16px).

  • 1rem = 16px (always)
  • 2rem = 32px (always)
  • 0.5rem = 8px (always)

The benefit: No compounding! 1rem is always 16px no matter how deeply nested.

Reboot's Key Principles

  • Use rem for spacing - Consistent, predictable sizing throughout
  • Avoid margin-top - Using only margin-bottom prevents spacing issues
  • Accessibility - When users change browser font size, rem-based layouts scale proportionally
  • Inherit fonts - Child elements inherit parent font settings unless specified otherwise

Page Defaults

The <html> and <body> elements are updated for better page-wide defaults:

  • Box-sizing - Set to border-box globally (including *::before and *::after)
  • Font-size - 16px assumed on <html>, 1rem on <body> for responsive scaling
  • Typography - Global font-family, font-weight, line-height, and color on <body>
  • Background - Default background-color: #fff for safety

Native Font Stack

zTheme uses a "native font stack" for optimum text rendering on every device and OS:

font-family:
  system-ui,              /* Cross-platform */
  -apple-system,          /* Safari (macOS/iOS) */
  "Segoe UI",             /* Windows */
  Roboto,                 /* Android */
  "Helvetica Neue", Arial,
  "Noto Sans",            /* Linux */
  "Liberation Sans",
  sans-serif,
  "Apple Color Emoji",    /* Emoji fonts */
  "Segoe UI Emoji",
  "Segoe UI Symbol",
  "Noto Color Emoji";

Note: The font stack includes emoji fonts, so many symbol characters will render as multi-colored pictographs. Appearance varies by browser/platform.

Forms

Various form elements have been rebooted for simpler base styles:

  • <fieldset>s have no borders, padding, or margin
  • <legend>s are styled as headings
  • <label>s are display: inline-block to allow margins
  • <textarea>s only resize vertically
  • <button>s have cursor: pointer when not disabled

→ For detailed form styling, see Forms

HTML5 [hidden] Attribute

HTML5's [hidden] attribute is improved to display: none !important; to prevent accidental overrides.

<input type="text" hidden>

Note: To toggle visibility without affecting document flow, use the .zD-none class instead.
