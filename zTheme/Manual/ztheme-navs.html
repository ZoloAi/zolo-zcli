Navs & Tabs

Flexible navigation components for building menus, tabs, pills, and content switchers with semantic HTML and accessible interactions.

Understanding Navs & Tabs

Navigation components are essential for organizing and switching between related content. Use them for:

  • Tabs: Switch between views of related content (settings sections, form steps)
  • Pills: Button-style navigation for dashboard sections or filters
  • Vertical Nav: Sidebar navigation for documentation or settings
  • Content Switching: Show/hide content without page reload
  • Sub-navigation: Secondary navigation within a page section
Note: For top-level site navigation (header menus), use Navbar instead. This component is for in-page navigation and content switching.

Fill and Justify

Long Text: This item is wider because it has more text. Compare with "A" which is narrow.

Fill (Proportional Growth)

With .zNav-fill, items grow proportionally to fill available space, but longer items stay wider. On mobile, if items don't fit, they scroll horizontally:

  • A
  • Long Text

Fill behavior: Items fill the entire width but maintain proportional sizing. "A" grows, "Long Text" is wider. No empty space!

Long Text: This item is wider than "A" because of its longer content, but both fill the available space proportionally.

â†‘ Click to compare. Items fill the space proportionally: "A" grows but "Long Text" is wider - no gaps!

Justified (Equal Width)

With .zNav-justified, all items have exactly equal width regardless of content. On mobile, if items would be too narrow, they scroll horizontally:

  • A
  • Long Text

Justified behavior: All items have EXACTLY the same width - perfectly symmetrical! See how "A" is as wide as "Long Text".

Long Text: This item is the exact same width as "A" despite having more text - that's justified layout!

Force Vertical Stacking (Optional)

For settings pages or special cases where vertical stacking is preferred, add .zNav-mobile-vertical:

<!-- Force vertical stacking on mobile -->
<ul class="zNav zNav-tabs zNav-mobile-vertical">
  <li class="zNav-item">
    <a class="zNav-link zActive" href="#">Settings</a>
  </li>
  <li class="zNav-item">
    <a class="zNav-link" href="#">Privacy</a>
  </li>
</ul>
Mobile Best Practice: Horizontal scrolling tabs are the industry standard for mobile. Only use vertical stacking for sidebar-style navigation or settings interfaces.

Best Practices

  • Use semantic HTML: Use <nav> for navigation, <ul> for related groups
  • Accessible markup: Include role="tablist", role="tab", role="tabpanel"
  • Active state: Always mark one item with .zActive and aria-current="page"
  • Disabled items: Use .zDisabled with tabindex="-1" and aria-disabled="true"
  • Keyboard navigation: Tab system includes arrow key navigation automatically
  • Content structure: Wrap tab panes in .zTab-content for proper styling
  • Mobile friendly: Tabs scroll horizontally automatically on mobile (768px and below)
  • Limit tabs: Keep to 5-7 tabs max; more than that, consider dropdowns or reorganizing
  • Short labels: Keep tab labels concise for better mobile experience
  • Test touch: Ensure tabs are easy to tap on mobile (min 44Ã—44px touch target)

Keyboard Navigation

The tab system automatically supports:

  • Tab - Move focus into/out of tab list
  • Arrow Left/Right - Navigate between tabs (horizontal)
  • Arrow Up/Down - Navigate between tabs (vertical)
  • Home - Jump to first tab
  • End - Jump to last tab
  • Enter/Space - Activate focused tab
