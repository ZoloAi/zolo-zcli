Badges

Small count and labeling components that scale to match their parent element's size.

Understanding Badges

Badges are compact visual indicators used to highlight new or important information. They're perfect for:

  • Counts: Unread messages, notifications, cart items
  • Labels: Status indicators, tags, categories
  • Highlights: "New", "Beta", "Pro" features
  • Metadata: Version numbers, dates, counts

Badges automatically scale to match the size of their parent element using relative font sizing (em units), making them adaptable to headings, buttons, and text of any size.

Accessibility: Badges may confuse screen reader users if context isn't clear. For example, a button that says "Notifications 4" is clear, but just "4" without context isn't. Use .zVisually-hidden to add extra context for assistive technologies when needed.

The Span Element

Badges are built using the <span> element—a generic inline container with no semantic meaning. Without styling, spans are just plain inline text:

Dashboard New

You have 12 unread messages.

Status: Online

<h3>Dashboard <span>New</span></h3>
<p>You have <span>12</span> unread messages.</p>
<p>Status: <span>Online</span></p>
Non-Semantic HTML: Unlike <article> or <button>, <span> has no inherent meaning—it's purely a styling hook. Use it when you need to style a portion of inline content without adding semantic context.

Enhanced Badges with .zBadge

The .zBadge class transforms plain spans into visually distinct badges. Compare this to the unstyled examples above:

Dashboard New

You have 12 unread messages.

Status: Online

<h3>Dashboard <span class="zBadge zBadge-success">New</span></h3>
<p>You have <span class="zBadge zBadge-danger">12</span> unread.</p>
<p>Status: <span class="zBadge zBadge-info">Online</span></p>

The transformation: .zBadge adds background color, padding, rounded corners, bold text, and relative sizing—turning plain text into eye-catching labels.

zTheme Pattern: The <span> element provides the inline structure (reboot = unstyled). The .zBadge utility class adds professional visual design. Separation of concerns!

Badge Variants

Use contextual variant classes to match badge colors to your message type. All 8 zTheme colors are available:

Primary Secondary Success Danger Warning Info Light Dark 
<span class="zBadge zBadge-primary">Primary</span>
<span class="zBadge zBadge-success">Success</span>
<span class="zBadge zBadge-danger">Danger</span>
<span class="zBadge zBadge-info">Info</span>
Color Meaning: Choose variants based on semantic meaning—.zBadge-success for positive states, .zBadge-danger for errors, .zBadge-warning for cautions. This helps all users understand context at a glance.

Badges in Buttons

Add badges to buttons to display counters or status indicators. Common for notification and messaging interfaces:

<button type="button" class="zBtn zBtn-primary">
  Notifications <span class="zBadge zBadge-light">4</span>
</button>
Screen Readers: In this example, "Notifications 4" is clear to all users. If the context isn't obvious, add hidden text: <span class="zVisually-hidden">unread messages</span>

Positioned Badges

Use positioning utilities to place badges as notification dots on buttons or icons. Great for subtle indicators:

<button type="button" class="zBtn zBtn-primary" style="position: relative;">
  Inbox
  <span style="position: absolute; top: 0; left: 100%; 
               transform: translate(-50%, -50%);" 
        class="zBadge zRounded-pill zBadge-danger">
    99+
    <span class="zVisually-hidden">unread messages</span>
  </span>
</button>

Note: The second example uses inline styles to create a simple notification dot without text—just a colored circle.

Pill Badges

Add .zRounded-pill to make badges more rounded with a larger border radius. Perfect for tags and labels:

<span class="zBadge zRounded-pill zBadge-success">Success</span>
<span class="zBadge zRounded-pill zBadge-info">Info</span>

Best Practices

  • Use sparingly: Too many badges create visual clutter—highlight only what's truly important
  • Keep text short: Badges are for quick glances—use 1-3 words max
  • Choose appropriate colors: Match badge variants to their semantic meaning (success = green, danger = red)
  • Don't replace buttons: Badges are informational, not interactive—use buttons for actions
  • Provide context: Ensure screen reader users understand the badge's meaning
  • Test contrast: Verify text is readable on the badge background (especially .zBadge-light)
  • Consider scale: Badges inherit parent size—test them in all contexts (headings, buttons, text)

.zBadge-dark Dark, high contrast badge

Sizing

Badges use relative sizing (font-size: 0.75em) and automatically scale to match their parent element. No additional size classes are needed.
