Z Index

Control element layering and stacking

What is Z-index?

The z-index property controls the vertical stacking order of positioned elements. Higher values appear in front of lower values.

Important: Z-index only works on positioned elements (position: relative, absolute, fixed, or sticky).

Z-Index Classes

Class Value Use Case
.zZ-n1 -1 Behind content
.zZ-0 0 Default stacking
.zZ-1 1 Slightly elevated
.zZ-2 2 More elevated
.zZ-3 3 Even more elevated
.zZ-10 10 High priority
.zZ-100 100 Very high
.zZ-1000 1000 UI overlays
.zZ-auto auto Reset to default

Component Layer System

Pre-defined z-index values for common UI components:

Class Value Component
.zZ-dropdown 1000 Dropdown menus
.zZ-sticky 1020 Sticky elements
.zZ-fixed 1030 Fixed navbars
.zZ-modal-backdrop 1040 Modal background overlay
.zZ-modal 1050 Modal dialogs
.zZ-popover 1060 Popovers
.zZ-tooltip 1070 Tooltips (highest)

Understanding Stacking Context

A stacking context is created when an element has:

  • Position relative, absolute, fixed, or sticky with z-index other than auto
  • Opacity less than 1
  • Transform, filter, or other CSS properties

Key Point: Z-index only compares elements within the same stacking context. A child can never appear above its parent's stacking context.

Best Practices

  • Use semantic layers - Prefer .zZ-modal over random numbers
  • Avoid z-index wars - Don't keep increasing values arbitrarily
  • Document layers - Create a z-index scale for your project
  • Test thoroughly - Check stacking in all scenarios
  • Remember position - Z-index needs a positioned element

CSS Reference

Defined in zPosition.css:

.zZ-1 { z-index: 1 !important; }
.zZ-2 { z-index: 2 !important; }
.zZ-3 { z-index: 3 !important; }

/* Component layers */
.zZ-dropdown { z-index: 1000 !important; }
.zZ-modal { z-index: 1050 !important; }
.zZ-tooltip { z-index: 1070 !important; }
