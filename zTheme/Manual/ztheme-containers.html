Containers

Center and constrain your content at different screen sizes

Understanding Containers

Containers do three things:

  • Constrain max-width - Prevent content from stretching too wide on large screens
  • Add consistent padding - Internal gutters (0.75rem) so content doesn't touch edges
  • Center themselves - Auto margins position the container horizontally in the viewport
‚úÖ Why constrain width? Professional websites limit content width because text lines wider than ~75 characters become hard to read. On a 1920px monitor, unconstrained text would stretch 150+ characters wide, causing eye strain and poor readability. Containers ensure optimal reading experience across all screen sizes.

üí° Which one to use?
  • Most pages: .zContainer (responsive at all breakpoints)
  • Mobile-first design: .zContainer-md or .zContainer-lg (fluid on mobile)
  • Full-bleed: .zContainer-fluid (hero sections, backgrounds)

‚úÖ Best Practices

  • Don't nest containers - Use one container per page section
  • Use with grid: Wrap .zRow and columns inside a container
  • Default choice: Start with .zContainer unless you need full-width

üîß Customize Max-Widths

Override container widths in your custom CSS (load after zTheme):

/* In your custom.css */
@media (min-width: 768px) {
  .zContainer {
    max-width: 800px;
  }
}

@media (min-width: 992px) {
  .zContainer {
    max-width: 1000px;
  }
}

Tip: Keep max-widths between 720px-1200px for optimal readability.
