Position Utilities

Configure element positioning, placement, and layering with comprehensive utilities.

Understanding Position

The position property determines how an element is positioned in the document. Combined with edge positioning (top/bottom/start/end) and z-index, you can create complex layouts.

üí° Bootstrap 5 Alignment: We use .zStart-* and .zEnd-* (RTL-aware) alongside legacy .zLeft-* and .zRight-* for backward compatibility!

Position Values

Five position values control how elements are placed and behave during scrolling:

.zPosition-static

Default behavior. Element follows normal document flow. Top/bottom/left/right and z-index have no effect.

Use case: Reset positioned elements back to default behavior

.zPosition-relative

Positioned relative to itself. Element remains in document flow but can be offset. Creates positioning context for absolute children.

Use case: Container for absolute/fixed children, slight adjustments

.zPosition-absolute

Removed from document flow. Positioned relative to nearest positioned ancestor (or viewport if none). Other elements ignore its space.

Use case: Overlays, dropdowns, tooltips, badges, notifications

.zPosition-fixed

Fixed to viewport. Removed from document flow. Always visible even when scrolling. Positioned relative to viewport.

Use case: Fixed headers/footers, floating action buttons, sticky notifications

.zPosition-sticky

Hybrid behavior. Acts relative until scroll threshold, then becomes fixed. Stays within parent container.

Use case: Sticky headers, table headers, sidebar navigation

Centering with Transform

Transform utilities perfectly center elements by shifting them back by 50% of their own size.

.zTranslate-middle

Perfect Center!

transform: translate(-50%, -50%) - Centers both X and Y axes

.zTranslate-middle-x

Centered X (Top)
Centered X (Bottom)

transform: translateX(-50%) - Centers only horizontally

.zTranslate-middle-y

Centered Y (Left)
Centered Y (Right)

transform: translateY(-50%) - Centers only vertically 

<!-- Perfect center -->
<div class="zPosition-absolute zTop-50 zStart-50 zTranslate-middle">
  Centered!
</div>

<!-- Center horizontally only -->
<div class="zPosition-absolute zTop-0 zStart-50 zTranslate-middle-x">
  Top, centered X
</div>

<!-- Center vertically only -->
<div class="zPosition-absolute zTop-50 zStart-0 zTranslate-middle-y">
  Left, centered Y
</div>

Position Helpers

Pre-built positioning patterns for common layouts like fixed headers and sticky navigation.

.zFixed-top

Fixed header: position: fixed + top: 0 + full width + z-index: 1030

Use case: Fixed navigation headers that stay visible while scrolling

.zFixed-bottom

Fixed footer: position: fixed + bottom: 0 + full width + z-index: 1030

Use case: Cookie notices, bottom toolbars, floating action buttons

.zSticky-top

Sticky header: position: sticky + top: 0 + z-index: 1020

Use case: Section headers, table headers that stick when scrolling

Responsive: .zSticky-{breakpoint}-top

Conditional sticky: Only sticky at specified breakpoint and above

Breakpoints: sm, md, lg, xl, xxl 

<!-- Fixed header -->
<nav class="zNavbar zFixed-top">...</nav>

<!-- Fixed footer/toolbar -->
<footer class="zFixed-bottom">...</footer>

<!-- Sticky header -->
<header class="zSticky-top">...</header>

<!-- Sticky only on desktop -->
<aside class="zSticky-lg-top">Sidebar</aside>

Z-Index Layering

Control element stacking order with z-index utilities.

Numeric Values

  • .zZ-n1 - Behind
  • .zZ-0 - Default
  • .zZ-1, .zZ-2, .zZ-3
  • .zZ-10, .zZ-100, .zZ-1000
  • .zZ-auto - Auto

Component Layers

  • .zZ-dropdown - 1000
  • .zZ-sticky - 1020
  • .zZ-fixed - 1030
  • .zZ-modal-backdrop - 1040
  • .zZ-modal - 1050
  • .zZ-popover - 1060
  • .zZ-tooltip - 1070

üí° Component Layers: Use semantic z-index classes like .zZ-modal instead of numeric values for consistent layering across your app!

Best Practices

  • Positioning context: Absolute elements need a positioned parent (relative/absolute/fixed)
  • Z-index layers: Use semantic classes (.zZ-modal) instead of arbitrary numbers
  • Fixed elements: May require padding on body to prevent content overlap
  • Sticky support: Check browser compatibility (IE11 doesn't support it)
  • Transform centering: Use .zTranslate-middle for perfect centering
  • RTL support: Prefer .zStart-* and .zEnd-* over .zLeft-* and .zRight-*
  • Mobile-first: Use responsive sticky (.zSticky-lg-top) for desktop-only stickiness

Accessibility

  • Fixed/absolute positioning: Can disrupt natural tab order for keyboard users
  • Hidden content: Content outside viewport may be inaccessible to screen readers
  • Focus management: Ensure focus moves logically, not by visual position
  • Skip links: Provide skip-to-content links with fixed headers
  • Badges: Use .zVisually-hidden for screen reader text in notification badges
  • Overlays: Trap focus within modals/dialogs, return focus on close
