Progress

Visualize task completion with Bootstrap-inspired progress bars featuring labels, backgrounds, stripes, and animations.

Understanding Progress Bars

Progress bars provide visual feedback about the completion status of operations. They consist of a wrapper (`.zProgress`) and an inner bar (`.zProgress-bar`) that sets the width based on progress value.

Key Features:

  • Accessible by default - Built-in ARIA attributes for screen readers
  • Smooth transitions - Width changes animate smoothly (0.6s ease)
  • Flexible styling - Colors, stripes, and height variants
  • Stacked bars - Multiple bars can show different segments
  • Indeterminate state - For operations with unknown duration (bonus feature)

üí° When to use progress bars:

  • File uploads/downloads - Show transfer progress
  • Form completion - Multi-step forms or onboarding
  • Loading states - Data processing or computation
  • Goal tracking - Fundraising, achievements, skill levels
  • Batch operations - Processing multiple items

Basic Progress Bar

Set the width of the `.zProgress-bar` using inline styles or width utilities. Always include ARIA attributes for accessibility.

0%
<div class="zProgress">
  <div class="zProgress-bar" 
       role="progressbar" 
       style="width: 25%" 
       aria-valuenow="25" 
       aria-valuemin="0" 
       aria-valuemax="100"></div>
</div>

Best Practices

‚úÖ DO:

  • Always include ARIA attributes (role, aria-valuenow, aria-valuemin, aria-valuemax)
  • Show percentage labels when space allows (helps users estimate time)
  • Update progress smoothly - let CSS transitions handle animation
  • Use colors meaningfully (success=complete, warning=caution, danger=error)
  • Consider indeterminate state for operations with unknown duration
  • Test with keyboard navigation and screen readers

‚ùå DON'T:

  • Use progress bars for binary states (use checkmarks or toggles instead)
  • Fake progress - users will notice if 90% takes as long as first 50%
  • Animate progress bar width manually (let CSS transitions do it)
  • Use too many colors in stacked bars (3-4 max for readability)
  • Forget to update ARIA values when progress changes
  • Use progress bars for very fast operations (< 1 second)

Accessibility

Progress bars are accessible by default when you include the proper ARIA attributes:

Required ARIA Attributes

  • role="progressbar" - Identifies the element as a progress bar
  • aria-valuenow - Current value (e.g., 75 for 75%)
  • aria-valuemin - Minimum value (usually 0)
  • aria-valuemax - Maximum value (usually 100)

Screen Reader Announcement

When a screen reader encounters a progress bar with proper ARIA attributes, it announces:

"Progress bar, 75 percent"

Make sure to update aria-valuenow via JavaScript as progress changes!

Reduced Motion

Animations (striped, indeterminate) automatically respect the prefers-reduced-motion CSS media query. Users who prefer reduced motion won't see animations.
