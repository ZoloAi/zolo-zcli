Collapse

Toggle the visibility of content with smooth animations using data attributes or JavaScript.

Understanding Collapse

The collapse component lets you show and hide content with a smooth height animation. It's perfect for:

  • FAQs: Hide long answers until users click questions
  • Expandable sections: Show more details on demand
  • Mobile navigation: Collapse menus on small screens
  • Forms: Show/hide advanced options

Unlike accordions (which allow only one section open), collapse is independent—multiple sections can be open at once.

Accessibility: Collapse respects prefers-reduced-motion and uses aria-expanded to tell screen readers whether content is visible or hidden.

Basic Example

Click the button to toggle visibility of the content below:

Collapsed Content

This content is hidden by default but revealed when you click the trigger button. The height animates smoothly from 0 to its full height.

You can put any content here: text, images, forms, even other components!

<button class="zBtn zBtn-primary" 
        data-bs-toggle="collapse" 
        data-bs-target="#collapseExample"
        aria-expanded="false"
        aria-controls="collapseExample">
  Toggle Content
</button>

<div class="zCollapse" id="collapseExample">
  <div class="zCard zCard-body">
    Your content here...
  </div>
</div>

How it works:

  • data-bs-toggle="collapse" — Tells zTheme this button triggers collapse
  • data-bs-target="#collapseExample" — Points to the element to show/hide
  • aria-expanded="false" — Accessibility: tells screen readers the state (auto-updated)
  • aria-controls="collapseExample" — Links control to target for screen readers

Link as Trigger

You can also use a link with href instead of a button:

Toggle via Link
This collapse is triggered by a link styled as a button! 
<a href="#collapseLinkExample" 
   class="zBtn zBtn-secondary" 
   data-bs-toggle="collapse" 
   role="button"
   aria-expanded="false">
  Toggle via Link
</a>

<div class="zCollapse" id="collapseLinkExample">
  Content here...
</div>
Semantic Note: When using <a> as a button, add role="button" for accessibility. Prefer <button> when possible.

Multiple Targets

A single button can toggle multiple elements by using a class selector:

Section 1

This is the first collapsible section.

Section 2

This is the second collapsible section.

<!-- Single button for multiple targets -->
<button data-bs-toggle="collapse" data-bs-target=".multi-collapse">
  Toggle Both
</button>

<!-- Both elements share the 'multi-collapse' class -->
<div class="zCollapse multi-collapse" id="section1">...</div>
<div class="zCollapse multi-collapse" id="section2">...</div>

Best Practices

  • Use semantic triggers: Prefer <button> over <a> for collapse triggers
  • Always include ARIA: Use aria-expanded and aria-controls for accessibility
  • Provide clear labels: Button text should indicate what will happen ("Show more", "Expand details")
  • Consider alternatives: For grouped collapsible sections, use Accordion instead
  • Keep content focused: Collapsed content should be secondary information, not critical
  • Mobile-friendly: Collapse is perfect for hiding long content on mobile devices
Tip: Collapse works great with Cards! Wrap collapsed content in .zCard for a polished look.

CSS Reference

Classes

Class Description
.zCollapse Base class for collapsible content (hidden by default)
.zShow Makes content visible (add for default open state)
.zCollapsing Applied during transition (managed by JavaScript)

Data Attributes

Attribute Required Description
data-bs-toggle="collapse" Yes Identifies element as a collapse trigger
data-bs-target Yes (or href) CSS selector for target element(s)
aria-expanded Recommended Accessibility: current state (true/false)
aria-controls Recommended Accessibility: ID of controlled element

Auto-Initialization

Collapse is automatically initialized by zTheme.js when the page loads. No additional JavaScript needed!

Auto-Initialization: Just add the data attributes and zTheme.js handles the rest. For dynamically added content, call window.zTheme.initCollapse() to re-initialize.
