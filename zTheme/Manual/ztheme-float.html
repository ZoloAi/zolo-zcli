Float Utilities

Toggle floats on any element, across any breakpoint, using responsive float utilities.

Understanding Float

Float utilities position elements to the left or right of their container, allowing content to wrap around them. This is useful for layouts like text wrapping around images or creating simple column layouts.

Basic Float

Three basic float values control element positioning:

.zFloat-start

Floated Start

This text wraps around the floated element on the left. Float start (left in LTR languages) positions the element on the left side of its container, allowing content to flow around it on the right. Perfect for wrapping text around images or creating sidebar layouts.

.zFloat-end

Floated End

This text wraps around the floated element on the right. Float end (right in LTR languages) positions the element on the right side of its container, allowing content to flow around it on the left. Commonly used for pull quotes or right-aligned images.

.zFloat-none

No Float Applied

This element is not floated and remains in the normal document flow. Use .zFloat-none to disable floating on an element (useful for removing inherited float behavior or responsive overrides). 

<!-- Float left (start) -->
<img src="..." class="zFloat-start" alt="...">
<p>Text wraps around the image...</p>

<!-- Float right (end) -->
<img src="..." class="zFloat-end" alt="...">
<p>Text wraps around the image...</p>

<!-- No float -->
<div class="zFloat-none">Block element</div>

Best Practices

  • Clear floats: Use .zClearfix on parent containers or clear: both after floated elements
  • Set width: Floated elements should usually have a defined width (percentage or fixed)
  • Add margin: Give floated elements margin to prevent text from touching them
  • Consider flexbox: For complex layouts, flexbox or grid may be more appropriate
  • Responsive first: Start with mobile (no float) and add floating at larger breakpoints
  • Test wrapping: Ensure text wraps naturally around floated elements at all screen sizes

üí° Tip: Float utilities are perfect for traditional document-style layouts (images in text, pull quotes) but Flexbox and Grid are better for modern app-style layouts with complex alignment needs.

Accessibility

  • Source order: Float changes visual order but not source order - keep content logical in HTML
  • Focus order: Ensure keyboard navigation follows logical flow, not visual placement
  • Images: Always include descriptive alt text for floated images
  • Mobile testing: Verify content makes sense when floats are removed on small screens
  • Screen readers: Remember screen readers follow HTML source order, not float layout
