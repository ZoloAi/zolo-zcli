Understanding Spinners

Spinners indicate the loading state of a component or page. They're lightweight, accessible, and built entirely with CSS animations‚Äîyou only need JavaScript to toggle their visibility.

When to use spinners:

  • API requests - Show loading while fetching data
  • Form submissions - Indicate processing on submit buttons
  • Page loads - Display while content is loading
  • File uploads - Show activity during upload
  • Background processes - Any asynchronous operation

‚ôø Accessibility: All spinners include role="status" and a nested <span class="zVisually-hidden">Loading...</span> for screen readers. Animations respect the prefers-reduced-motion media query.

Best Practices

‚úÖ DO:

  • Always include role="status" for accessibility
  • Add <span class="zVisually-hidden">Loading...</span> for screen readers
  • Use spinners to indicate actual loading/processing states
  • Match spinner color to your UI context (use inline styles or currentColor)
  • Disable buttons with spinners to prevent double-submission
  • Choose spinner type based on context (border for most, grow for emphasis)

‚ùå DON'T:

  • Use spinners as decorative elements‚Äîthey indicate loading states only
  • Show spinners for instant operations (< 200ms)
  • Use too many spinners at once (confusing for users)
  • Forget to hide spinners when loading completes
  • Use spinners without accessible text for screen readers
  • Make spinners too large (distracting) or too small (invisible)

Accessibility

Spinners are accessible by default when you follow the proper structure:

Required Attributes

  • role="status" - Identifies the element as a status indicator
  • .zVisually-hidden - Provides text for screen readers ("Loading...")
  • aria-hidden="true" - On decorative spinners (inside buttons with visible text)

Reduced Motion

All spinners automatically slow down for users who prefer reduced motion (prefers-reduced-motion: reduce media query). Animation duration increases from 0.75s to 1.5s for a gentler experience.
