Understanding Toasts

Toasts are lightweight notification pop-ups designed to mimic mobile push notifications. They appear temporarily in a corner of the screen to inform users about actions, updates, or system messages without interrupting their workflow.

Perfect for:

  • Success confirmations - "File uploaded successfully"
  • Error notifications - "Connection lost, retrying..."
  • Status updates - "3 new messages received"
  • Background operations - "Export complete"
  • Undo actions - "Item deleted. Undo?"

‚ö†Ô∏è Toasts vs Alerts:

  • Toasts: Temporary, auto-dismiss, non-blocking, corner-positioned
  • Alerts: Persistent, inline with content, require user action to dismiss

Use toasts for transient messages, alerts for important warnings that need attention.

Data Attributes

Configure toast behavior with these data attributes:

Attribute Type Default Description
data-bs-animation boolean true Apply CSS fade transition
data-bs-autohide boolean true Automatically hide the toast
data-bs-delay number 5000 Delay before auto-hide (milliseconds)
data-bs-dismiss string "toast" Add to button to enable dismiss

Best Practices

‚úÖ DO:

  • Keep messages short and scannable (1-2 sentences max)
  • Use color coding to convey meaning (green=success, red=error)
  • Position consistently (usually top-right or bottom-right)
  • Include a dismiss button for manual closing
  • Use appropriate ARIA attributes for accessibility
  • Set reasonable auto-hide delays (3-7 seconds)
  • Stack toasts vertically for multiple notifications

‚ùå DON'T:

  • Use toasts for critical errors (use modals or alerts instead)
  • Make toasts auto-hide too quickly (users need time to read)
  • Show too many toasts at once (3 max recommended)
  • Put important actions in auto-hiding toasts
  • Use toasts for long-form content (use modals)
  • Block the main content area with toasts
  • Forget to test with screen readers

Accessibility

Toasts must be accessible to all users, especially screen reader users:

Required ARIA Attributes

  • role="alert" - For errors and critical messages
  • role="status" - For informational messages
  • aria-live="assertive" - For urgent messages (interrupts)
  • aria-live="polite" - For non-urgent messages (waits for pause)
  • aria-atomic="true" - Ensures entire toast is read

Important Considerations

  • Auto-hide timing: Give users enough time to read (minimum 5 seconds)
  • Keyboard access: If autohide: false, ensure dismiss button is keyboard accessible
  • Live regions: Toasts should exist in the DOM before being shown
  • Focus management: Toasts don't receive focus (by design, to avoid interrupting workflow)

Reduced Motion

Fade animations are automatically disabled for users who prefer reduced motion via the prefers-reduced-motion media query.
