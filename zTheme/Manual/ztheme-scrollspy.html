Scrollspy

Automatically highlight navigation links based on scroll position in the viewport.

Understanding Scrollspy

Scrollspy automatically updates navigation or list group components based on scroll position. As users scroll through your page, the corresponding navigation link becomes active, showing them where they are in the content.

Perfect for:

  • Documentation pages - Help users navigate long technical docs
  • Landing pages - One-page sites with sections (About, Services, Contact)
  • Article sidebars - Table of contents that follows the reader
  • Step-by-step guides - Show progress through multi-step processes

How it works:

  1. Add data-bs-spy="scroll" to the scrollable element
  2. Add data-bs-target="#navbar" pointing to your nav
  3. Navigation links must use anchor tags (<a href="#section1">)
  4. Target sections must have matching IDs (<div id="section1">)
  5. JavaScript automatically adds/removes .zActive class

Features

Scrollspy comes with several powerful features out of the box:

  • Automatic activation: Links highlight automatically based on scroll position
  • Smooth scrolling: Clicking nav links smoothly scrolls to sections
  • Nested navigation: Works with multi-level nav structures
  • Dropdown support: Activates parent dropdowns when child items are active
  • Flexible targets: Works with any scrollable container, not just body

All of these features are built-in and require no additional configuration.

Setup

Setting up scrollspy is straightforward. Here's what you need:

  1. Add data attributes to your scrollable container
  2. Create a navigation with anchor links
  3. Create sections with matching IDs
  4. Let JavaScript handle the rest!

The system automatically detects when sections enter the viewport and updates the navigation accordingly.

No manual event listeners or complex configuration needed.

Data Attributes

Configure scrollspy behavior with these data attributes:

Attribute Value Description
data-bs-spy "scroll" Required. Enables scrollspy on the element.
data-bs-target CSS selector Required. Targets the navigation element (e.g., "#navbar").
data-bs-offset Number (pixels) Optional. Offset from top when calculating position. Default: 10
tabindex "0" Recommended. Makes container keyboard accessible.

Requirements

For scrollspy to work correctly, follow these requirements:

‚úÖ Required Elements

  1. Bootstrap nav or list group: Must use .zNav or .zList-group
  2. Position relative: Spied element needs position: relative (usually <body>)
  3. Anchor links: Navigation must use <a href="#section"> tags
  4. Matching IDs: Target sections need corresponding id attributes

üéØ For Scrollable Containers

If using a container other than <body>:

  • Set a height on the container
  • Add overflow-y: scroll
  • Include tabindex="0" for keyboard access
  • Consider position: relative or position: sticky for the nav

Best Practices

‚úÖ DO:

  • Use scrollspy for content with clear sections (3+ headings)
  • Keep navigation labels short and descriptive
  • Ensure sections have enough content to be scrollable
  • Use scroll-margin-top CSS to account for fixed headers
  • Test with keyboard navigation (tabindex="0")
  • Consider sticky navigation for better usability

‚ùå DON'T:

  • Use scrollspy on short pages (less than 2 screen heights)
  • Have too many navigation items (5-8 max for readability)
  • Forget to add IDs to all target sections
  • Use very short sections (they'll activate/deactivate too quickly)
  • Nest scrollspy elements (only one level supported)
  • Forget accessibility attributes (tabindex, ARIA labels)

Accessibility

Make scrollspy accessible to all users:

Keyboard Navigation

  • Add tabindex="0": Makes scrollable containers keyboard accessible
  • Arrow keys: Users can scroll with keyboard arrows
  • Tab key: Lets users tab through navigation links
  • Enter key: Activates links to jump to sections

Screen Readers

  • Use semantic HTML: <nav>, <h2>, <section>
  • Add ARIA labels: aria-label="Table of contents" on nav
  • Provide skip links: Allow jumping past navigation
  • Announce changes: activate.bs.scrollspy event can trigger announcements

Smooth Scrolling

Smooth scrolling is enabled via CSS (scroll-behavior: smooth) and automatically disabled for users who prefer reduced motion via the prefers-reduced-motion media query.

JavaScript Events

Scrollspy fires a custom event when a new item becomes active:

activate.bs.scrollspy

Fires on the scroll element whenever a new navigation item becomes active.

const scrollSpyEl = document.querySelector('[data-bs-spy="scroll"]');

scrollSpyEl.addEventListener('activate.bs.scrollspy', function(e) {
  console.log('New section activated:', e.detail.relatedTarget);
  // e.detail.relatedTarget contains the activated link element
});

üí° Use case: Track user engagement by logging which sections users visit, or update a progress indicator showing how far through the content they've scrolled.

CSS Classes

Scrollspy uses existing navigation classes with automatic .zActive state:

Class Purpose
.zActive Applied to current nav link by JavaScript
.zNav-link Navigation links (from zNav.css)
.zList-group-item List group items (from zListGroup.css)

No additional CSS classes needed - scrollspy works with existing zTheme navigation components!
