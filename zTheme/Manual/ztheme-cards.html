Cards

Flexible and extensible content containers with multiple variants and options for headers, footers, images, and contextual styling

Understanding Cards

What are Cards?

Cards are flexible content containers that can hold a wide variety of content, including text, images, links, and more. They're one of the most versatile UI components because they:

  • Group related content: Keep information organized and scannable
  • Provide visual hierarchy: Clear boundaries between different content sections
  • Work everywhere: Perfect for product listings, blog posts, user profiles, dashboards
  • Flexible layout: Can be arranged in grids, columns, or custom layouts
Semantic HTML: Cards use semantic elements like <article>, <section>, or <div> depending on their content. For standalone content (blog posts, products), use <article>. For grouped UI elements, use <div>.

Semantic HTML Structure

Before adding card styling, understand the semantic HTML foundation. Cards can use different HTML elements depending on their content:

Option 1: Using <div> (Most Common)

Use <div> for purely visual grouping - no semantic meaning, just layout:

Dashboard Widget

Current system status: All services running normally.

View details 
<div>
  <h3>Dashboard Widget</h3>
  <p>Current system status...</p>
  <a href="#">View details</a>
</div>
Notice: This is completely unstyled semantic HTML. No borders, no padding, no background - just browser defaults. This is true reboot.

Option 2: Using <article> (Semantic)

Use <article> for standalone content that could exist independently (blog posts, products, news items):

Product Update

New features released: dark mode, enhanced search, improved performance.

Read more 
<article>
  <h3>Product Update</h3>
  <p>New features released...</p>
  <a href="#">Read more</a>
</article>
Semantic Difference: While <div> and <article> look identical visually (both unstyled), <article> tells search engines and screen readers "this is standalone content." Choose based on meaning, not appearance.

When to use each:

  • <div>: UI components, dashboards, grouped elements (no SEO/accessibility benefit needed)
  • <article>: Blog posts, products, news items (helps search engines and screen readers)
  • <section>: Thematic grouping of content within a page
Truth: Most developers just use <div> for everything, and that's okay! But when you have standalone content (like a blog post card or product card), using <article> helps search engines understand your page better. The HTML element defines meaning, zTheme classes define visual style.

Enhanced Cards with .zCard

The .zCard class transforms plain semantic HTML into a professional card component. It works on any element (<div>, <article>, <section>):

Product Update

New features have been released including dark mode, enhanced search, and improved performance.

Read more 
<article class="zCard" style="width: 18rem;">
  <div class="zCard-body">
    <h5 class="zCard-title">Product Update</h5>
    <p class="zCard-text">New features have been released...</p>
    <a href="#" class="zCard-link">Read more</a>
  </div>
</article>

The transformation: Compare this to the unstyled examples above. .zCard adds:

  • White background - Contrasts with page background
  • Subtle shadow - Creates depth and "lifts" the card
  • Light border - Defines clear boundaries
  • Rounded corners - Modern, friendly appearance
  • Proper padding - Content isn't cramped (via .zCard-body)
  • Typography spacing - Professional spacing between title, text, and links
zTheme Pattern: Semantic HTML defines meaning (reboot = unstyled). zTheme classes like .zCard add professional visual design. Separation of concerns!

Card Layouts with Grid

Use zTheme's grid system to create responsive card layouts. This is the modern, recommended approach:

Product One

High-quality product with excellent reviews and fast shipping.

View Details
Product Two

Premium features with lifetime warranty and customer support.

View Details
Product Three

Budget-friendly option without compromising on quality.

View Details 
<div class="zRow zRow-cols-1 zRow-cols-md-3 zG-4">
  <div class="zCol">
    <div class="zCard">
      <img src="..." class="zCard-img-top" alt="Product 1">
      <div class="zCard-body">
        <h5 class="zCard-title">Product One</h5>
        <p class="zCard-text">High-quality product...</p>
        <a href="#" class="zBtn zBtn-primary">View Details</a>
      </div>
    </div>
  </div>
  <!-- Repeat for more cards -->
</div>
Modern Approach: Bootstrap 5 (and zTheme) recommend using the grid system (.zRow and .zCol-*) instead of the older .zCard-group for card layouts. The grid provides better responsive control and flexibility.

Best Practices

  • Keep cards focused: Each card should contain one clear piece of content or action
  • Use consistent sizing: Cards in a grid should have similar content lengths or use .h-100 for equal heights
  • Provide context: Use titles, headers, or footers to clarify what the card contains
  • Accessible images: Always include descriptive alt attributes on card images
  • Mobile-first: Test card layouts on mobile - cards stack naturally on smaller screens
  • Call-to-action: Include clear buttons or links for interactive cards
  • Semantic HTML: Use <article> for standalone content, <section> for grouped content
