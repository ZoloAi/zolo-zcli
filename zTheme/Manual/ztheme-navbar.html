Navbar

Responsive navigation headers for site-wide navigation with branding, links, dropdowns, and mobile toggling.

Understanding Navbar

A navbar is a site-wide navigation component that appears at the top (or bottom) of your pages. It's essential for:

  • Site Navigation: Primary links to main sections (Home, Products, About, Contact)
  • Branding: Logo and company name for consistent identity
  • Responsive Layout: Collapses into a mobile menu on smaller screens
  • Utility Links: Login, search, cart, or user profile access
  • Fixed Position: Stays visible during scrolling for easy access
Note: For in-page navigation and content switching (tabs, pills), use Navs & Tabs instead. Navbar is specifically for site-wide, persistent navigation.

The Nav Element

HTML provides the semantic <nav> element for navigation sections. Our reboot provides minimal, clean styling for navigation elements:

Brand Home Features About 
<nav>
  <a href="/">Brand</a>
  <a href="/">Home</a>
  <a href="/features">Features</a>
  <a href="/about">About</a>
</nav>

What the reboot provides:

  • Clean, semantic HTML structure
  • Basic link styling
  • Consistent spacing
  • Accessibility foundation

Best Practices

  • Use semantic HTML: Always use <nav> for navigation sections
  • Clear branding: Make your brand/logo a link to the home page
  • Limit links: Keep navbar to 3-7 top-level items for clarity
  • Active state: Mark the current page with .active and aria-current="page"
  • Mobile first: Test toggle functionality on mobile breakpoints
  • Use dropdowns: Group related links in dropdowns instead of cluttering the navbar
  • Consistent placement: Keep navbar position consistent across your site
  • Test keyboard: Ensure all links are accessible via keyboard (Tab key)
  • Skip link: Consider adding a "Skip to main content" link for screen readers

Keyboard Navigation

Navbar links automatically support keyboard navigation:

  • Tab - Move between links and toggle button
  • Enter or Space - Activate link or toggle
  • Esc - Close expanded mobile menu (if open)

Theme Modifiers

Class Effect
.zNavbar-light Light background theme (gray text on light bg)
.zNavbar-dark Dark background theme (white text on dark bg)

Positioning Modifiers

Class Effect
.zNavbar-fixed-top Fixed to top of viewport (position: fixed; top: 0;)
.zNavbar-fixed-bottom Fixed to bottom of viewport (position: fixed; bottom: 0;)
.zNavbar-sticky-top Sticks to top when scrolled (position: sticky; top: 0;)

Dropdown Classes

Class Purpose
.zNav-dropdown Dropdown container on nav item
.zNav-dropdown-menu Dropdown menu container (hidden by default)
.zNav-dropdown-item Individual dropdown menu item

Data Attributes

Attribute Description
data-bs-toggle="collapse" Enables toggle behavior on button
data-bs-target="#id" Target element ID to collapse/expand
aria-expanded Indicates if menu is expanded (true/false)
aria-controls ID of element controlled by button
aria-current="page" Marks current/active page link
