Form Layout

Structure your forms with vertical stacking, horizontal layouts, or multi-column grids.

Understanding Form Layout

Form layout determines how form controls, labels, and buttons are arranged on the page. zTheme provides no default styling for the <form> element itself—instead, you use the grid system and spacing utilities to create the layout you need.

Common layouts:

  • Vertical (default): Labels above inputs, everything stacks—best for mobile, simple forms
  • Horizontal: Labels beside inputs—compact, good for settings/preferences
  • Multi-column: Multiple fields per row—efficient for long forms (checkout, profile)
Mobile-First: All form controls (.zInput, .zSelect, etc.) are display: block and width: 100% by default, so they stack vertically. Use the grid system to create side-by-side layouts for larger screens.

When to use each: Vertical for mobile-first/simple forms, horizontal for compact settings, multi-column for data-heavy forms (addresses, checkout).

Vertical Forms (Default)

With no layout classes, forms stack vertically. Use spacing utilities like .zmb-3 (margin-bottom) to separate fields:

<form>
  <div class="zmb-3">
    <label for="email" class="zLabel">Email Address</label>
    <input type="email" class="zInput" id="email">
  </div>
  <div class="zmb-3">
    <label for="password" class="zLabel">Password</label>
    <input type="password" class="zInput" id="password">
  </div>
  <button type="submit" class="zBtn zBtn-info">Sign In</button>
</form>

Multi-Column Forms

Use the grid system (.zRow, .zCol-*) to create multi-column layouts. Add .zG-3 for gutters:

<form class="zRow zG-3">
  <div class="zCol-md-6 zmb-3">
    <label for="firstName" class="zLabel zmb-2">First Name</label>
    <input type="text" class="zInput" id="firstName">
  </div>
  <div class="zCol-md-6 zmb-3">
    <label for="lastName" class="zLabel zmb-2">Last Name</label>
    <input type="text" class="zInput" id="lastName">
  </div>
  <div class="zCol-12 zmb-3">
    <label for="address" class="zLabel zmb-2">Street Address</label>
    <input type="text" class="zInput" id="address">
  </div>
  <!-- More fields... -->
  <div class="zCol-12">
    <button type="submit" class="zBtn zBtn-info">Save Address</button>
  </div>
</form>
Key spacing pattern: Use .zmb-2 on labels (tight label-to-input), .zmb-3 on columns (field group separation), and .zG-3 for horizontal gaps between side-by-side fields. This creates proper hierarchy in both mobile (stacked) and desktop (side-by-side) views.
Mobile behavior: On small screens, .zCol-md-6 becomes full-width (stacks vertically). On medium screens and up, fields appear side-by-side. This is mobile-first responsive design!

Horizontal Forms

Place labels and inputs side-by-side using .zRow for each field. Labels typically use .zCol-sm-2 or .zCol-sm-3:

<form>
  <div class="zRow zmb-3">
    <label for="email" class="zCol-sm-3 zLabel">Email</label>
    <div class="zCol-sm-9">
      <input type="email" class="zInput" id="email">
    </div>
  </div>
  
  <div class="zRow zmb-3">
    <label for="password" class="zCol-sm-3 zLabel">Password</label>
    <div class="zCol-sm-9">
      <input type="password" class="zInput" id="password">
    </div>
  </div>
  
  <!-- Checkbox with offset to align with inputs -->
  <div class="zRow zmb-3">
    <div class="zCol-sm-9 zOffset-sm-3">
      <div class="zCheck">
        <input class="zCheck-input" type="checkbox" id="remember">
        <label class="zCheck-label" for="remember">Remember me</label>
      </div>
    </div>
  </div>
</form>
Tip: Add padding-top: 0.5rem to labels to vertically center them with inputs. Use .zOffset-sm-3 to align elements (like checkboxes/buttons) with the input column.

Best Practices

Use Proper Spacing Hierarchy

Multi-column forms need two levels of spacing:

  • .zmb-2 on labels - tight connection to input (0.5rem)
  • .zmb-3 on columns - field group separation (1rem)
  • .zG-3 on form - horizontal gaps between side-by-side fields
<form class="zRow zG-3">
  <div class="zCol-md-6 zmb-3">
    <label class="zLabel zmb-2">First Name</label>
    <input type="text" class="zInput">
  </div>
</form>

Start Mobile-First

Always design forms for vertical stacking (mobile), then enhance for larger screens:

  • ✅ .zmb-3 on columns ensures good mobile spacing
  • ✅ .zCol-md-6 creates side-by-side on desktop
  • ❌ Don't rely only on .zG-3—it doesn't control vertical stacking

Group Related Fields

Use <fieldset> and <legend> to group related fields for accessibility:

<fieldset>
  <legend>Shipping Address</legend>
  <!-- Address fields here -->
</fieldset>

<fieldset>
  <legend>Payment Information</legend>
  <!-- Payment fields here -->
</fieldset>

Choose the Right Layout

  • Vertical: Login, signup, simple contact forms (≤5 fields)
  • Multi-column: Addresses, checkout, long forms (>10 fields)
  • Horizontal: Settings, preferences, admin panels (compact space)
