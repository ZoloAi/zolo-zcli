Accordion

Organize content into collapsible panels that expand and collapse with smooth animations

Understanding Accordions

What are Accordions?

Accordions are vertically stacked panels that expand to reveal hidden content. Only one panel is typically open at a time, making them perfect for:

  • FAQs: Frequently Asked Questions
  • Documentation: Organized by topic or category
  • Product features: Detailed specifications
  • Complex forms: Breaking long forms into sections

Why Use Accordions?

  • Save space: Show only relevant content
  • Reduce cognitive load: Focus on one section at a time
  • Organize content: Natural hierarchy for related information
  • Mobile-friendly: Perfect for smaller screens
Accessibility: Accordions use semantic HTML and ARIA attributes to ensure screen readers announce expanded/collapsed states. Users can navigate with keyboard (Tab, Enter, Space).

Benefits of Native HTML

  • ✅ No JavaScript: Works out of the box
  • ✅ Fully accessible: Screen readers understand it natively
  • ✅ Keyboard friendly: Tab and Enter work automatically
  • ✅ Lightweight: Zero dependencies
  • ✅ Progressive enhancement: Works even if CSS/JS fails

Limitations

  • ❌ Each panel independent: Can't auto-close others
  • ❌ Limited styling: Browser default appearance varies
  • ❌ No animations: Instant expand/collapse (no smooth transition)
  • ❌ No grouping: Can't enforce "only one open at a time"
When to use native vs zAccordion:
Use <details> for simple, independent collapsible sections (like a single FAQ item).
Use .zAccordion for grouped panels with consistent styling, auto-close behavior, and smooth animations.

JavaScript Implementation

Automatic Initialization

Accordions work automatically when you include zTheme.js (already loaded in base.html). No additional JavaScript needed!

✨ Auto-Initialization: Just add the .zAccordion class to your container, and zTheme.js automatically handles all click events, state management, and ARIA attributes. It works just like form validation!

How It Works (Under the Hood)

For developers curious about the implementation, here's how zTheme.js handles accordions:

// Inside zTheme.js - automatically runs on page load
function initAccordion() {
    const accordions = document.querySelectorAll('.zAccordion');
    
    accordions.forEach(accordion => {
        const buttons = accordion.querySelectorAll('.zAccordion-button');
        
        buttons.forEach(button => {
            button.addEventListener('click', function() {
                const panel = button.closest('.zAccordion-item')
                                   .querySelector('.zAccordion-collapse');
                
                const isOpen = !panel.classList.contains('zCollapsed');
                
                // Close all panels in this accordion
                accordion.querySelectorAll('.zAccordion-collapse')
                         .forEach(p => p.classList.add('zCollapsed'));
                accordion.querySelectorAll('.zAccordion-button')
                         .forEach(b => {
                             b.classList.add('zCollapsed');
                             b.setAttribute('aria-expanded', 'false');
                         });
                
                // Toggle the clicked panel
                if (isOpen) {
                    panel.classList.add('zCollapsed');
                    button.classList.add('zCollapsed');
                    button.setAttribute('aria-expanded', 'false');
                } else {
                    panel.classList.remove('zCollapsed');
                    button.classList.remove('zCollapsed');
                    button.setAttribute('aria-expanded', 'true');
                }
            });
        });
    });
}

Manual Re-initialization

If you dynamically add accordions after page load (e.g., via AJAX), re-initialize them:

// After dynamically adding an accordion
window.zTheme.initAccordion();
Note: When zTheme.js is integrated into zBifrost client (future enhancement), accordion initialization will happen automatically even for dynamically rendered content.

Best Practices

Use Semantic Headings

Always wrap accordion buttons in proper heading tags (<h2>, <h3>, etc.) for semantic structure and SEO:

  • ✅ <h3 class="zAccordion-header"><button>...</button></h3>
  • ❌ <div class="zAccordion-header"><button>...</button></div>

Keep Content Concise

Accordion panels should contain focused, digestible content. If a panel is too long, consider:

  • Breaking it into multiple panels
  • Using nested accordions (sparingly)
  • Linking to a dedicated page for detailed content

Consider Initial State

Think about which panel(s) should be open by default:

  • First panel open: Good for FAQs or getting started guides
  • All closed: Good for optional details or advanced options
  • Specific panel open: Based on user context or URL anchor

Test Keyboard Navigation

Ensure users can:

  • Tab: Navigate between accordion buttons
  • Enter/Space: Toggle panels
  • Screen readers: Hear "expanded" or "collapsed" states

CSS Reference

Class Hierarchy

Class Purpose
.zAccordion Container for accordion items
.zAccordion-item Individual accordion panel
.zAccordion-header Heading wrapper (use <h2>-<h6>)
.zAccordion-button Clickable button to toggle panel
.zAccordion-collapse Collapsible content wrapper
.zAccordion-body Actual content inside panel
.zCollapsed Applied to closed panels (toggle state)
.zAccordion-flush Modifier for borderless, edge-to-edge style
