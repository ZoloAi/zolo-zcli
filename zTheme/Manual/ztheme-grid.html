The 12 Columns System

How It Works

The 12 Columns grid system uses a standard three-layer structure:
Container ‚Üí Row ‚Üí Columns.

After wrapping your content in a zContainer, you create zRows to hold zColumns. Each row divides into 12 equal columns, giving you maximum layout flexibility.

Columns

Columns live inside rows and hold your content. Each column takes a portion of the 12 available units. You can use equal-width columns (auto-sized), fixed-width columns (specify exact sizes), or responsive columns (different sizes at different breakpoints).

Column Classes

The pattern is simple: .zCol-{breakpoint}-{width} where width is 1-12.

Remember: breakpoints are screen widths where your layout changes. Here are all the column classes:

  • .zCol-1 through .zCol-12 - Mobile (default)
  • .zCol-sm-1 through .zCol-sm-12 - Small (‚â•576px)
  • .zCol-md-1 through .zCol-md-12 - Tablet (‚â•768px)
  • .zCol-lg-1 through .zCol-lg-12 - Desktop (‚â•992px)
  • .zCol-xl-1 through .zCol-xl-12 - Large (‚â•1200px)
  • .zCol-xxl-1 through .zCol-xxl-12 - XL (‚â•1400px)

No number? Use .zCol, .zCol-md, etc. for auto-width (equal distribution)

Column classes can look confusing at first. Here's how to read them:

.zCol-md-6

  • ‚Ü≥ .zCol = Column
  • ‚Ü≥ -md = At medium breakpoint (‚â•768px)
  • ‚Ü≥ -6 = Take 6 out of 12 units (half width)

No breakpoint? .zCol-6 means "6 units wide on ALL screens (mobile-first default)"

No number? .zCol-md means "auto-width at medium breakpoint"

Best Practices

  • Container ‚Üí Row ‚Üí Columns - Always nest in this order
  • Add up to 12 - Column numbers should total 12 per row
  • Mobile-first - Start with .zCol-12, add breakpoint classes
  • Nest properly - Row ‚Üí Column ‚Üí Row (not Row ‚Üí Row)
