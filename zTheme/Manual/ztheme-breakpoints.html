Breakpoints

Responsive breakpoints for different screen sizes

Core Concepts

  • Control when layouts adapt to different screen sizes
  • Use media queries with min-width to apply styles at specific breakpoints
  • Mobile-first approach: start with mobile styles, layer on styles for larger screens

Understanding Display Classes

zTheme uses zD-* classes to control visibility at different breakpoints:

  • zD = Display utility prefix
  • -block = Show element (display: block)
  • -none = Hide element (display: none)
  • -md- = At medium breakpoint (‚â•768px) and up
  • -lg- = At large breakpoint (‚â•992px) and up

Replace block with other display values:

  • .zD-none ‚Üí Hide element
  • .zD-block ‚Üí Display as block
  • .zD-inline ‚Üí Display inline
  • .zD-inline-block ‚Üí Display inline-block
  • .zD-flex ‚Üí Display as flexbox

Combine with breakpoints: .zD-md-flex, .zD-lg-inline, etc.

Understanding Media Queries

A @media query applies styles only when the screen is a certain size:

  • @media (min-width: 768px) = At 768px and larger
  • @media (min-width: 992px) = At 992px and larger
  • No media query = Applies to all screen sizes (mobile first)

The CSS:

.my-element {
  font-size: 14px;
}

@media (min-width: 768px) {
  .my-element {
    font-size: 16px;
  }
}

@media (min-width: 992px) {
  .my-element {
    font-size: 18px;
  }
}
üìñ How it works:
  • Mobile (xs, sm): Font is 14px
  • Tablet (md): Screen ‚â•768px ‚Üí Font becomes 16px
  • Desktop (lg+): Screen ‚â•992px ‚Üí Font becomes 18px

Each larger breakpoint overrides the previous size. Styles cascade up! üìà

üí° Use CSS Variables

Instead of hardcoding pixel values, use zTheme variables from zBreakpoints.css:

@media (min-width: var(--breakpoint-md)) {
  /* Your styles for tablet+ */
}

Available variables: --breakpoint-sm (576px), --breakpoint-md (768px), --breakpoint-lg (992px), --breakpoint-xl (1200px), --breakpoint-xxl (1400px)

üîß Override Breakpoints Globally

Want different breakpoint values for your entire app? Override them in your custom CSS file:

/* In your custom.css (load after zTheme) */
:root {
  --breakpoint-md: 800px;  /* Change tablet from 768px to 800px */
  --breakpoint-lg: 1024px; /* Change desktop from 992px to 1024px */
}

Result: All zTheme classes and your @media queries using these variables will automatically use your custom values! üéØ

Note: Load your custom CSS after zTheme CSS files to ensure your overrides take precedence.
