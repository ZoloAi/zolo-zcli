List Group

Flexible component for displaying a series of content—from simple lists to complex custom content.

Understanding List Groups

List groups are versatile containers perfect for:

  • Lists of items: Users, products, tasks, notifications
  • Navigation menus: Sidebar navigation with active states
  • Settings panels: Grouped configuration options
  • Activity feeds: Timeline or notification lists
  • Step indicators: Multi-step processes or wizards

Unlike plain <ul> or <ol> elements, list groups provide consistent styling, active states, color variants, and can function as clickable navigation.

Basic List Group

Create a list group with .zList-group on a <ul> and .zList-group-item on each <li>:

</ul>

Links and Buttons

Make list groups actionable by using <a> or <button> with .zList-group-item-action:

With Links

The current link item A second link item A third link item A fourth link item A disabled link item 
<div class="zList-group">
  <a href="#" class="zList-group-item zList-group-item-action zActive">
    Current item
  </a>
  <a href="#" class="zList-group-item zList-group-item-action">
    Another item
  </a>
</div>

With Buttons

Semantic choice: Use <button> for actions (delete, edit), <a> for navigation (links to pages). Buttons support the native disabled attribute; links require .zDisabled and tabindex="-1".

As Navigation (Tabs)

List groups can function as tab navigation with data-bs-toggle="list":

Home Profile Messages Settings

Home

This is the home tab content. Click different items in the list to switch tabs.

Profile

This is the profile tab content. List groups make excellent sidebar navigation!

Messages

This is the messages tab content. Each tab is independently activated.

Settings

This is the settings tab content. Perfect for settings panels or dashboards!

<!-- List navigation -->
<div class="zList-group" role="tablist">
  <a class="zList-group-item zList-group-item-action zActive" 
     data-bs-toggle="list" 
     href="#home" 
     role="tab">Home</a>
  <a class="zList-group-item zList-group-item-action" 
     data-bs-toggle="list" 
     href="#profile" 
     role="tab">Profile</a>
</div>

<!-- Tab content -->
<div class="zTab-content">
  <div class="zTab-pane zActive zShow" id="home" role="tabpanel">
    Home content...
  </div>
  <div class="zTab-pane" id="profile" role="tabpanel">
    Profile content...
  </div>
</div>
Auto-Initialization: Just add data-bs-toggle="list" and zTheme.js handles tab switching automatically!

Best Practices

  • Use semantic HTML: Use <ul>/<ol> for lists, <div> for navigation
  • Show current state: Always mark one item as .zActive for navigation
  • Disable unavailable: Use .zDisabled instead of hiding options
  • Keep lists focused: Group related items together
  • Use flush in cards: .zList-group-flush looks great in .zCard
  • Combine with badges: Show counts or notifications with .zBadge
  • Mobile-friendly: Horizontal lists stack on mobile (responsive variants)
  • Consider alternatives: For tabbed navigation, also see Button Groups

CSS Reference

List Group Classes

Class Description
.zList-group Base container (use on <ul>, <ol>, or <div>)
.zList-group-item Individual list item
.zList-group-item-action Makes item clickable with hover effect
.zActive Marks current/selected item
.zDisabled Grays out unavailable item

Variants

Class Description
.zList-group-flush Remove borders and rounded corners
.zList-group-numbered Add automatic numbering (use with <ol>)
.zList-group-horizontal Horizontal layout (all breakpoints)
.zList-group-horizontal-sm Horizontal on small+ screens
.zList-group-horizontal-md Horizontal on medium+ screens
.zList-group-horizontal-lg Horizontal on large+ screens

Contextual Colors

Class Description

.zList-group-item-secondary Gray background
.zList-group-item-success Green background
.zList-group-item-info Cyan background
.zList-group-item-warning Yellow background
.zList-group-item-danger Red background
.zList-group-item-light Light gray background
.zList-group-item-dark Dark gray background

Tab Classes (Navigation)

Class Description
.zTab-content Container for tab panes
.zTab-pane Individual tab content panel
.zFade Add fade transition to tab panes
.zShow Shows the active tab pane

Data Attributes (Tab Navigation)

Attribute Required Description
data-bs-toggle="list" Yes Enables tab navigation functionality
href or data-bs-target Yes ID of target tab pane (e.g., #home)
role="tab" Recommended Accessibility: identifies as tab control
aria-selected Recommended Accessibility: current state (auto-managed)
