Interaction Utilities

Control how users interact with content through text selection and pointer events.

Understanding Interaction Utilities

Interaction utilities control three key aspects of user interaction:

  • Text Selection: Control whether and how text can be selected/copied
  • Pointer Events: Control whether elements respond to mouse/touch interactions
  • Cursor Styles: Change the mouse cursor to provide visual feedback

Text Selection

Change the way content is selected when users interact with it. Try selecting the text in each box!

.zUser-select-all

Click anywhere in this paragraph to select ALL text at once. Try it! When you click, the entire paragraph is automatically selected. This is useful for code snippets, license keys, or any content users should copy in full.

Use case: Code blocks, API keys, license codes, copy-to-clipboard content

.zUser-select-auto

This paragraph has default selection behavior. You can select text normally by clicking and dragging. This is the standard behavior you're used to. Use this class to restore default selection on elements that inherit non-standard behavior.

Use case: Restoring default behavior, overriding inherited restrictions

.zUser-select-none

This text cannot be selected! Try to highlight this text - you can't! The cursor changes but nothing gets selected. This is useful for UI labels, buttons, or decorative text that shouldn't be copied.

Use case: UI labels, button text, navigation items, decorative elements 

<!-- Select all text on click -->
<code class="zUser-select-all">npm install zolo-zcli</code>

<!-- Default selection -->
<p class="zUser-select-auto">Normal paragraph text</p>

<!-- Prevent selection -->
<nav class="zUser-select-none">
  <a href="#">Home</a>
  <a href="#">About</a>
</nav>

Pointer Events

Control whether elements respond to mouse, touch, and stylus interactions. Try clicking the links below!

.zPe-none

This link cannot be clicked - Try clicking it, nothing happens! The cursor changes to show it's disabled.

‚ö†Ô∏è Accessibility: Add tabindex="-1" and aria-disabled="true" to prevent keyboard access and inform screen readers!

.zPe-auto

This link can be clicked (default behavior) - This is normal link behavior. Click it to see an alert!

Inherited .zPe-none

This entire box has .zPe-none, so this link can't be clicked because it inherits the property. However, this link has .zPe-auto and CAN be clicked (it overrides the parent)! 

<!-- Disable pointer events -->
<a href="#" class="zPe-none" tabindex="-1" aria-disabled="true">
  Disabled link
</a>

<!-- Enable pointer events (default or override) -->
<a href="#" class="zPe-auto">Active link</a>

<!-- Parent with disabled events -->
<div class="zPe-none">
  <a href="#" tabindex="-1" aria-disabled="true">Disabled</a>
  <a href="#" class="zPe-auto">But this works!</a>
</div>

Best Practices

Text Selection

  • Don't overuse: Most content should remain selectable for accessibility
  • Use .zUser-select-all: For API keys, code snippets, license keys
  • Use .zUser-select-none: For UI chrome (buttons, nav, controls), not content
  • Allow content selection: Users should be able to copy article text, quotes, etc.

Pointer Events

  • Add keyboard attributes: Always include tabindex="-1" with .zPe-none
  • Add ARIA: Include aria-disabled="true" for screen readers
  • Visual feedback: Disabled elements should look disabled (opacity, cursor)
  • Consider HTML: For forms, use disabled attribute instead
  • Loading states: Temporarily disable buttons during async operations
  • Overlay blocks: Use pointer-events on overlays to prevent background clicks

Cursor Styles

  • Match interactivity: Use .zCursor-pointer on clickable non-link elements
  • Combine with state: Pair .zCursor-not-allowed with .zPe-none for disabled items
  • Loading feedback: Use .zCursor-wait during async operations
  • Drag affordance: Use .zCursor-grab on draggable items, .zCursor-grabbing while dragging
  • Zoom indicators: Show .zCursor-zoom-in on zoomable images/maps
  • Don't mislead: Only use pointer cursor on truly interactive elements

üí° Remember: .zPe-none only blocks pointer interactions (mouse/touch). It doesn't prevent keyboard access! Always add tabindex="-1" and consider using JavaScript for full disablement.

Accessibility Considerations

Text Selection

  • Screen readers: Can still read non-selectable text (good!)
  • Right-click menus: May not work on non-selectable content
  • User frustration: Unexpected selection behavior can confuse users
  • Search & find: Browser find (Ctrl+F) still works on non-selectable text

Pointer Events

  • Keyboard users: .zPe-none alone doesn't block keyboard - add tabindex="-1"
  • Screen readers: Add aria-disabled="true" to communicate state
  • Forms: Use disabled HTML attribute on form controls instead
  • Focus management: Disabled elements should not receive focus
  • JavaScript: May be needed to fully prevent all interactions
