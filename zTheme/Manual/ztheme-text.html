Text Utilities

Control text alignment, wrapping, transformation, sizing, weight, and styling.

Understanding Text Utilities

Text utilities control how text appears and behaves. Use these utilities to align text, control wrapping, adjust size and weight, and apply transformations without custom CSS.

Text Alignment

Align text within its container using RTL-aware utilities:

.zText-start - Start aligned (left in LTR, right in RTL)

.zText-center - Center aligned (same in both directions)

.zText-end - End aligned (right in LTR, left in RTL) 

<p class="zText-start">Start aligned</p>
<p class="zText-center">Center aligned</p>
<p class="zText-end">End aligned</p>

Text Transform

Change text case with transform utilities:

.zText-lowercase - THIS BECOMES this

.zText-uppercase - this becomes THIS

.zText-capitalize - this becomes This 

<p class="zText-lowercase">MAKE lowercase</p>
<p class="zText-uppercase">make UPPERCASE</p>
<p class="zText-capitalize">capitalize each word</p>

üí° Capitalize only changes first letter: .zText-capitalize transforms "hELLO wORLD" to "HELLO WORLD" (only first letter of each word).

Line Height

Control spacing between lines of text:

.zLh-1

Tight line height (1). Multiple lines show very close spacing between them.

.zLh-sm

Small line height (1.25). Multiple lines show slightly closer spacing.

.zLh-base

Base line height (1.5). Default spacing, good for reading.

.zLh-lg

Large line height (2). Multiple lines show generous spacing. 

<p class="zLh-1">Tight spacing</p>
<p class="zLh-sm">Small spacing</p>
<p class="zLh-base">Normal spacing (default)</p>
<p class="zLh-lg">Loose spacing</p>

Best Practices

  • Readability first: Use .zLh-base or .zLh-lg for body text
  • Hierarchy with size: Use font size utilities to establish visual hierarchy
  • Weight for emphasis: Use .zFw-bold sparingly for key information
  • Transform cautiously: Uppercase can be hard to read in long passages
  • Truncate with care: Always provide full text in title attribute or tooltip
  • Monospace for code: Always use .zFont-monospace for code snippets
  • Center sparingly: Center align works best for headings and short text
  • Break for URLs: Use .zText-break for long URLs or email addresses

‚ö†Ô∏è Don't mix too many: Combining too many text utilities can make content hard to read. Keep it simple and consistent!

Accessibility

  • Sufficient contrast: Ensure text color contrasts well with background (WCAG AA minimum)
  • Line height for readability: Use .zLh-base or larger for body text
  • Don't rely on uppercase alone: Screen readers may not announce case changes
  • Avoid full justification: Not provided as it reduces readability
  • Truncated text: Provide full content in alt text or expandable section
  • Italic sparingly: Can be harder to read for dyslexic users
  • Monospace sizing: May need larger font size for readability
