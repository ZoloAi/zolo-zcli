Understanding Colored Links

Colored links provide semantic meaning through color while maintaining proper hover and focus states. Unlike text color utilities, these link classes include interactive states that darken on hover/focus, providing clear visual feedback to users.

üí° Centralized Colors:

All link colors use CSS variables from css_vars.css, ensuring consistent theming across your entire application. Change the color variables once, and all colored links update automatically!

üé® Color System:

  • Primary: var(--color-primary) - Brand identity
  • Secondary: var(--color-secondary) - Supporting brand color
  • Success: var(--color-success) - Positive actions/states
  • Info: var(--color-info) - Informational content
  • Warning: var(--color-warning) - Cautionary messages
  • Danger: var(--color-error) - Destructive actions/errors
  • Dark/Light: var(--color-dark/light) - Neutral tones

Best Practices

‚úÖ DO:

  • Use colors to convey semantic meaning (success = positive, danger = destructive)
  • Keep link text descriptive ("View profile" instead of "Click here")
  • Test hover states on both light and dark backgrounds
  • Ensure color contrast meets WCAG AA standards (4.5:1 ratio)
  • Use consistent colors for similar actions across your app
  • Combine with icons for additional context

‚ùå DON'T:

  • Don't rely on color alone - Use text, icons, or context too (accessibility!)
  • Don't use .zLink-light on light backgrounds (insufficient contrast)
  • Don't overuse multiple colors in one paragraph (visual noise)
  • Don't use danger links for non-destructive actions
  • Don't forget hover states matter for UX clarity
  • Don't use warning links on white backgrounds without testing contrast

‚ö†Ô∏è Color Contrast Warnings:

  • .zLink-light: Only use on dark backgrounds!
  • .zLink-warning: May need contrast testing on some backgrounds
  • Always test: Use browser dev tools to check contrast ratios

Accessibility

Color is Not Enough

While colored links are visually helpful, never rely on color alone to convey meaning. Users with color blindness or screen readers won't perceive the color. Always include:

  • Descriptive text: "Delete account" instead of just colored "Click here"
  • Icons: Combine colors with symbols (‚úì, ‚ö†Ô∏è, ‚úï)
  • Context: Surrounding text should explain the action
  • ARIA labels: For icon-only links

Keyboard Navigation

  • Tab: Navigate between links
  • Enter: Activate link
  • Focus visible: All links show focus state
  • Screen readers: Announce link text (ensure it's descriptive!)

Color Contrast Standards

All zTheme link colors meet WCAG AA contrast requirements on appropriate backgrounds. However, always test in your specific context:

  • WCAG AA: 4.5:1 contrast ratio (normal text)
  • WCAG AAA: 7:1 contrast ratio (enhanced)
  • Tool: Use browser DevTools to check contrast
