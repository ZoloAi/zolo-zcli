Modal

Dialog overlays for lightboxes, user notifications, and custom content that demand user attention.

Understanding Modals

Modals are dialog boxes that overlay the page, perfect for:

  • Confirmations: "Are you sure you want to delete?"
  • Forms: Login, registration, or data entry without leaving the page
  • Alerts & notifications: Important messages requiring acknowledgment
  • Image lightboxes: Full-size image viewing
  • Multi-step wizards: Guided processes in a focused context

Modals lock the page scroll, dim the background (backdrop), and capture user focus until dismissed—making them perfect for content that requires immediate attention.

Use sparingly: Modals interrupt user flow. Reserve them for important moments. For less critical content, consider Collapse or Dropdowns.

Basic Modal

A modal has three main parts: header (title + close button), body (content), and footer (action buttons):

<!-- Button trigger -->
<button type="button" class="zBtn zBtn-primary" 
        data-bs-toggle="modal" 
        data-bs-target="#basicModal">
  Launch demo modal
</button>

<!-- Modal -->
<div class="zModal zFade" id="basicModal" tabindex="-1" aria-hidden="true">
  <div class="zModal-dialog">
    <div class="zModal-content">
      <div class="zModal-header">
        <h5 class="zModal-title">Modal title</h5>
        <button type="button" class="zBtn-close" data-bs-dismiss="modal" aria-label="Close"></button>
      </div>
      <div class="zModal-body">
        <p>Woohoo, you're reading this text in a modal!</p>
      </div>
      <div class="zModal-footer">
        <button type="button" class="zBtn zBtn-secondary" data-bs-dismiss="modal">Close</button>
        <button type="button" class="zBtn zBtn-primary">Save changes</button>
      </div>
    </div>
  </div>
</div>
Auto-Initialization: Just add data-bs-toggle="modal" and zTheme.js handles everything: backdrop, animations, focus, keyboard navigation, and body scroll lock!

Static Backdrop

By default, clicking outside the modal closes it. Use data-bs-backdrop="static" to prevent this:

<div class="zModal zFade" id="staticModal" 
     data-bs-backdrop="static" 
     data-bs-keyboard="false" 
     tabindex="-1">
  <div class="zModal-dialog">
    <div class="zModal-content">
      <div class="zModal-header">
        <h5 class="zModal-title">Modal title</h5>
        <button type="button" class="zBtn-close" data-bs-dismiss="modal"></button>
      </div>
      <div class="zModal-body">
        <p>I will not close if you click outside me. Don't even try ESC key.</p>
      </div>
      <div class="zModal-footer">
        <button type="button" class="zBtn zBtn-secondary" data-bs-dismiss="modal">Close</button>
        <button type="button" class="zBtn zBtn-primary">Understood</button>
      </div>
    </div>
  </div>
</div>

Tip: Use static backdrop for critical confirmations (like "Delete account?") where you want to ensure the user consciously dismisses the modal.

Vertical Positioning

Control where the modal appears vertically on the screen:

<!-- Default: Top aligned with margin -->
<div class="zModal-dialog">...</div>

<!-- Vertically centered -->
<div class="zModal-dialog zModal-dialog-centered">...</div>

<!-- Bottom aligned -->
<div class="zModal-dialog zModal-dialog-bottom">...</div>

Animation Options

Choose from different animation styles to control how your modal enters the screen:

Class Effect Best For
.zFade Slide down from top (+ fade) Default, general purpose
.zSlideUp Slide up from bottom (+ fade) Mobile sheets, bottom actions
.zZoomIn Zoom/scale from center (+ fade) Image lightboxes, focus attention
.zSlideLeft Slide in from right (+ fade) Side panels, next/forward actions
.zSlideRight Slide in from left (+ fade) Side panels, back/previous actions
.zFadeOnly Pure opacity fade (no movement) Minimal, subtle transitions 
<!-- Slide down (default) -->
<div class="zModal zFade" id="myModal">...</div>

<!-- Slide up -->
<div class="zModal zSlideUp" id="myModal">...</div>

<!-- Zoom in -->
<div class="zModal zZoomIn" id="myModal">...</div>

<!-- Fade only (no movement) -->
<div class="zModal zFadeOnly" id="myModal">...</div>

<!-- No animation -->
<div class="zModal" id="myModal">...</div>
Accessibility: All animations respect prefers-reduced-motion and will be disabled for users who request reduced motion in their system settings.

Scrollable Content

For long content, add .zModal-dialog-scrollable to make only the body scroll while keeping the header and footer fixed in view:

Scrollable vs Default: Without .zModal-dialog-scrollable, the entire modal scrolls behind the backdrop. With it, only the body scrolls while header and footer remain visible—better UX for forms and long content!

Height Control

By default, scrollable modals take up most of the viewport. Control the height with size modifiers:

Class Height Best For
.zModal-scrollable-sm 400px (300px body) Short lists, simple forms
.zModal-scrollable-md 550px (450px body) Standard forms, moderate content
.zModal-scrollable-lg 700px (600px body) Long forms, detailed content
.zModal-scrollable-xl 850px (750px body) Very long content, data tables
(none) ~Full viewport Maximum content visibility 
<!-- Full height scrollable (default) -->
<div class="zModal-dialog zModal-dialog-scrollable">...</div>

<!-- Fixed height scrollable -->
<div class="zModal-dialog zModal-dialog-scrollable zModal-scrollable-md">...</div>

<!-- Custom height with CSS variable -->
<div class="zModal-dialog zModal-dialog-scrollable" style="--modal-height: 500px">...</div>

Fullscreen Modal

Use .zModal-fullscreen for modals that take up the entire viewport:

<!-- Always fullscreen -->
<div class="zModal-dialog zModal-fullscreen">...</div>

<!-- Fullscreen on small screens only -->
<div class="zModal-dialog zModal-fullscreen-sm-down">...</div>
<div class="zModal-dialog zModal-fullscreen-md-down">...</div>
<div class="zModal-dialog zModal-fullscreen-lg-down">...</div>

Mobile-first: Responsive fullscreen classes are perfect for mobile experiences—fullscreen on phones, centered dialog on tablets/desktops.

Best Practices

  • Always include a close button: Give users a clear way out (data-bs-dismiss="modal")
  • One modal at a time: Avoid nesting or stacking modals—it's confusing
  • Clear modal titles: Use descriptive .zModal-title for context
  • Keep it focused: Modals should have a single purpose
  • Mobile considerations: Test on small screens; consider .zModal-fullscreen-sm-down
  • Destructive actions: Use data-bs-backdrop="static" for confirmations
  • Accessibility: Include aria-labelledby, aria-hidden, and tabindex="-1"
  • Keyboard navigation: ESC closes modals (unless data-bs-keyboard="false")
  • Don't overuse: Excessive modals frustrate users—consider alternatives first

Accessibility

Modals are complex for screen readers. Follow these guidelines:

Required Attributes

<div class="zModal zFade" 
     id="myModal" 
     tabindex="-1" 
     aria-labelledby="myModalLabel" 
     aria-hidden="true">
  <div class="zModal-dialog">
    <div class="zModal-content">
      <div class="zModal-header">
        <h5 class="zModal-title" id="myModalLabel">Modal Title</h5>
        <button type="button" class="zBtn-close" 
                data-bs-dismiss="modal" 
                aria-label="Close"></button>
      </div>
      ...
    </div>
  </div>
</div>
  • tabindex="-1" - Allows focus on the modal container
  • aria-labelledby - Links to the modal title for screen readers
  • aria-hidden="true" - Hides modal from screen readers when closed (auto-managed by JS)
  • aria-label="Close" - Describes close button for screen readers

Focus management: When opened, focus moves to the first focusable element. When closed, focus returns to the trigger button.

Modal Structure

Class Description
.zModal Outer modal container (fixed overlay)
.zModal-dialog Centers and sizes the modal
.zModal-content White box containing header/body/footer
.zModal-header Header with title and close button
.zModal-title Modal title text
.zModal-body Main content area
.zModal-footer Footer with action buttons
.zModal-backdrop Dark background overlay (auto-generated by JS)

Size Modifiers

Class Description
.zModal-sm Small modal (300px)
.zModal-lg Large modal (800px)
.zModal-xl Extra large modal (1140px)
.zModal-fullscreen Full viewport width/height
.zModal-fullscreen-sm-down Fullscreen on small screens
.zModal-fullscreen-md-down Fullscreen on medium and below
.zModal-fullscreen-lg-down Fullscreen on large and below

Positioning Modifiers

Class Description
.zModal-dialog-centered Vertically center the modal
.zModal-dialog-bottom Align modal to bottom
.zModal-dialog-scrollable Make modal body scrollable (header/footer fixed)

Scrollable Height Control

Class Description
.zModal-scrollable-sm Small fixed height (400px / 300px on mobile)
.zModal-scrollable-md Medium fixed height (550px / 400px on mobile)
.zModal-scrollable-lg Large fixed height (700px / 500px on mobile)
.zModal-scrollable-xl Extra large fixed height (850px / 600px on mobile)

Note: Height control classes only work with .zModal-dialog-scrollable

Animation Classes

Class Description
.zFade Slide down from top + fade (default/recommended)
.zSlideUp Slide up from bottom + fade
.zZoomIn Zoom/scale from center + fade
.zSlideLeft Slide in from right + fade
.zSlideRight Slide in from left + fade
.zFadeOnly Pure opacity fade (no movement)
.zShow Show modal (auto-managed by JS)

Data Attributes

Attribute Required Description
data-bs-toggle="modal" Yes On trigger button to enable modal
data-bs-target Yes ID of target modal (e.g., #myModal)
data-bs-dismiss="modal" No On buttons to close modal
data-bs-backdrop="static" No Prevent closing on backdrop click
data-bs-keyboard="false" No Prevent closing with ESC key
