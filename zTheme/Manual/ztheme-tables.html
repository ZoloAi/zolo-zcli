Tables

Display data in organized rows and columns with semantic HTML tables and optional styling.

The Table Element

The <table> element is semantic HTML for displaying tabular data in rows and columns. Tables tell screen readers and search engines that your content is structured data, making them important for accessibility and SEO.

Table Structure

A proper table uses these semantic elements:

  • <table> - The container for all table content
  • <thead> - The header section (column labels)
  • <tbody> - The body section (data rows)
  • <tfoot> - The footer section (optional, for totals/summaries)
  • <tr> - A table row
  • <th> - A header cell (labels, usually bold)
  • <td> - A data cell (actual content)
  • <caption> - A table title/description (optional but helpful for accessibility)

Semantic importance: Always use <table> for tabular data, never for layout. Screen readers announce "table with X rows and Y columns" which helps users navigate your data.

Here's a table using just semantic HTML (no styling classes). The reboot intentionally provides minimal styling - no borders, no padding, just clean structure. This prevents zTheme from interfering with third-party widgets that use tables.

Let's look at a simple employee roster table. We use <thead> for column headers and <tbody> for the data rows:

ID First Name Last Name Department
101 Sarah Chen Engineering
102 Marcus Johnson Design
103 Elena Rodriguez Marketing 
<table>
  <thead>
    <tr>
      <th scope="col">ID</th>
      <th scope="col">First Name</th>
      <th scope="col">Last Name</th>
      <th scope="col">Department</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">101</th>
      <td>Sarah</td>
      <td>Chen</td>
      <td>Engineering</td>
    </tr>
    ...
  </tbody>
</table>

Why scope matters: Imagine you're blind and using a screen reader. When you hear "Chen", what does that mean? A first name? Last name? Department? Without context, data is meaningless.

The scope attribute solves this:

  • scope="col" - "This header labels everything below it" (like "Last Name" labels the entire column)
  • scope="row" - "This header labels everything next to it" (like "101" labels that entire row)

Now when the screen reader reads "Chen", it can say: "Last Name: Chen, Employee ID 101" - the data has context!

Opt-In Styling with .zTable

zTheme's table styling is opt-in. Add the .zTable class to get enhanced styling: better spacing, full width, consistent borders, and proper cell padding.

Here's the same employee roster with .zTable applied - notice the difference:

ID First Name Last Name Department
101 Sarah Chen Engineering
102 Marcus Johnson Design
103 Elena Rodriguez Marketing

  <thead>
    <tr>
      <th scope="col">ID</th>

      <td>Chen</td>

</table>

What .zTable adds: Full width, consistent cell padding, proper borders, margin below the table, and a foundation for modifier classes like .zTable-striped or .zTable-hover.

Vertical Alignment

Table cells in <thead> are always vertically aligned to the bottom. Table cells in <tbody> inherit their alignment from <table> and are aligned to the top by default. Use vertical align classes to re-align where needed.

Paginated Tables

For large datasets, combine tables with pagination controls to improve performance and usability. Show a subset of rows and let users navigate through pages.

ID Name Email Department Status
Showing 1-10 of 50 employees 
<!-- Table with striped and hover -->
<table class="zTable zTable-striped zTable-hover">
  <thead>
    <tr>
      <th scope="col">ID</th>
      <th scope="col">Name</th>
      <th scope="col">Email</th>
      <th scope="col">Department</th>
      <th scope="col">Status</th>
    </tr>
  </thead>
  <tbody id="tableBody">
    <!-- Rows generated dynamically -->
  </tbody>
</table>

<!-- Pagination Controls -->
<nav class="zPagination">
  <ul class="zPagination-list">
    <li class="zPage-item">
      <a class="zPage-link" onclick="goToPage(1)">Previous</a>
    </li>
    <li class="zPage-item zActive">
      <a class="zPage-link">1</a>
    </li>
    <!-- More page numbers -->
    <li class="zPage-item">
      <a class="zPage-link" onclick="goToPage(2)">Next</a>
    </li>
  </ul>
</nav>

üí° Best Practices for Paginated Tables:

  • Show 10-25 rows per page (10 for mobile, 25 for desktop)
  • Display total count and current range ("Showing 11-20 of 150")
  • Use server-side pagination for datasets with 1000+ rows
  • Preserve sort order and filters when changing pages
  • Consider infinite scroll as an alternative for mobile

Responsive Tables

Responsive tables allow tables to be scrolled horizontally with ease. Make any table responsive across all viewports by wrapping a .zTable with .zTable-responsive. Or, pick a maximum breakpoint with which to have a responsive table up to by using .zTable-responsive{-sm|-md|-lg|-xl|-xxl}.

Always Responsive

Across every breakpoint, use .zTable-responsive for horizontally scrolling tables.
