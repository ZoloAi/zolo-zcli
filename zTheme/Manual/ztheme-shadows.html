Understanding Shadows

Shadows create depth and elevation in your UI. They help establish visual hierarchy by making elements appear to float above the page at different heights. Subtle shadows create gentle separation, while larger shadows suggest interactive elements or important content.

üí° Material Design: Shadows communicate elevation. Higher elevations (larger shadows) typically indicate more important or interactive elements like buttons, cards, and modals.

üí° Notice the difference: Outward shadows make elements float above the surface. Inset shadows make elements look carved into the surface, like pressed buttons or input fields!

When to Use Each Shadow Type

Outward Shadows (Elevation)

Use when you want elements to appear to float above the page surface:

  • .zShadow-none - Flat design, remove inherited shadows, minimalist interfaces
  • .zShadow-sm - Subtle cards, list items, images, gentle separation
  • .zShadow - Default cards, panels, content boxes, most common use
  • .zShadow-lg - Featured content, dropdowns, popovers, important elements
  • .zShadow-xl - Modals, overlays, dialogs, maximum prominence

Inset Shadows (Depression)

Use when you want elements to appear carved into the page surface:

  • .zShadow-inset - Pressed buttons, active states, input fields, subtle wells
  • .zShadow-inset-lg - Content wells, recessed containers, embedded sections
  • .zShadow-inset-xl - Deep depressions, dramatic recessed areas, sunken panels

üí° Rule of Thumb: Outward shadows = interactive/important content that should stand out. Inset shadows = pressed/inactive states or content areas that should recede. Don't mix them on the same element!

<!-- Outward: Cards, buttons, dropdowns -->
<div class="zCard zShadow">Floating card</div>
<button class="zBtn zBtn-primary zShadow-sm">Click Me</button>
<div class="zModal zShadow-xl">Important modal</div>

<!-- Inset: Pressed states, input fields, wells -->
<button class="zBtn zShadow-inset">Pressed</button>
<input class="zForm-control zShadow-inset" />
<div class="zShadow-inset-lg">Content well</div>

Best Practices

  • Consistent elevation: Use shadow sizes to indicate element importance consistently
  • Interactive elements: Buttons and clickable cards should have shadows (.zShadow or .zShadow-sm)
  • Modals/overlays: Use .zShadow-xl for maximum elevation and visual separation
  • Subtle by default: Start with .zShadow-sm and increase only when needed
  • Hover effects: Increase shadow on hover to show interactivity
  • Performance: Shadows are GPU-accelerated but many can impact performance
  • Light backgrounds: Shadows work best on light backgrounds; adjust on dark themes
  • Border alternative: Use shadows instead of borders for modern, clean designs

‚ö†Ô∏è Don't overuse: Too many shadows can make a design feel cluttered. Use them strategically to highlight important elements and establish clear hierarchy!

Accessibility

  • Don't rely on shadows alone: Ensure sufficient color contrast without shadows
  • Focus indicators: Combine shadows with border/outline for keyboard focus
  • Reduced motion: Shadow transitions should respect prefers-reduced-motion
  • Low vision: Shadows may not be visible to all users - use multiple visual cues
  • Print styles: Consider removing/reducing shadows in print media
