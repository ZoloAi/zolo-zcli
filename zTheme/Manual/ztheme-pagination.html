Pagination

Navigate through pages of content with customizable pagination controls for search results, product listings, and data tables.

Understanding Pagination

Pagination breaks large content into multiple pages, making it easier to browse and improving performance. It's essential for:

  • Search Results: Navigate through hundreds or thousands of results
  • Product Listings: Browse e-commerce catalogs page by page
  • Data Tables: View large datasets in manageable chunks
  • Blog Archives: Navigate through posts chronologically
  • API Responses: Handle paginated data from backend services
Best Practice: Always show the current page number and total pages when possible. Users should know where they are and how much content remains.

<script>
  let currentPage = 1;
  const itemsPerPage = 6;
  const totalItems = 30;
  
  function renderPage(page) {
    // Calculate which items to show
    const start = (page - 1) * itemsPerPage;
    const end = start + itemsPerPage;
    
    // Update product grid
    productGrid.innerHTML = products.slice(start, end)
      .map(product => `<div>${product}</div>`)
      .join('');
    
    // Update pagination controls
    updatePagination(page);
  }
  
  function goToPage(page) {
    if (page < 1 || page > totalPages) return;
    currentPage = page;
    renderPage(page);
  }
</script>

Ellipsis Pattern: Show first page, last page, current page, and 1-2 pages on either side of current. Use "..." for gaps. This keeps pagination compact while providing context.

Best Practices

  • Always include Previous/Next: Users expect these controls
  • Show current page: Use .zActive class and aria-current="page"
  • Disable at boundaries: Previous on page 1, Next on last page
  • Use ellipsis for many pages: Don't show all 100 pages, use "..."
  • Consider total count: Show "Page 5 of 20" for context
  • Keep it accessible: Use semantic HTML (<nav>, aria-label)
  • Mobile-friendly: Test touch targets on small screens
  • Server-side pagination: For large datasets, paginate on backend
  • Preserve filters: Include query params in pagination URLs

    </li>
  </ul>
</nav>

Keyboard Navigation

Pagination links automatically support keyboard navigation:

  • Tab - Move to next page link
  • Shift + Tab - Move to previous page link
  • Enter - Activate the focused link
  • Disabled links are skipped in tab order (tabindex="-1")

Screen Reader Support

  • Use aria-label on <nav> to describe purpose
  • Use aria-current="page" to announce current page
  • Use aria-disabled="true" on disabled links
  • Use aria-hidden="true" on decorative symbols (¬´ ¬ª)
  • Provide descriptive aria-label on Previous/Next

CSS Reference

Base Classes

Class Description
.zPagination Main container (nav wrapper, centered by default)
.zPagination-list Flex container for page items (ul element)
.zPage-item Individual page wrapper (li element)
.zPage-link Clickable link/button with hover states

State Modifiers

Class Effect
.zActive Current page (blue background, white text)
.zDisabled Non-clickable state (grayed out, no pointer events)

Size Modifiers

Class Effect

Class Effect
