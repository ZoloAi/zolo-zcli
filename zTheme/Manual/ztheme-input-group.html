Input Group

Combine inputs with text labels, buttons, or dropdowns to create powerful form controls.

Understanding Input Groups

Input groups are compositional patterns that extend form controls by attaching contextual labels or actions. Built with <div> and <span> wrappers, they work WITH semantic form elements (<input>, <button>, <select>) to create accessible, meaningful interfaces.

This pattern improves usability by:

  • Adding context: @ symbols for usernames, $ for prices, domain names for URLs
  • Reducing fields: Combine related inputs (first/last name, price with currency)
  • Quick actions: Attach buttons for search, submit, or clear
  • Visual grouping: Show which elements work together
Accessibility: Input groups rely on proper semantic markup. Use <label> outside the group (linked via for/id), <button> for actions (not <span>!), and aria-describedby* to connect inputs to prefix/suffix text. Screen readers will announce: "Email Address, edit text, described by domain, @company.com"‚Äîgiving full context.

When to use: Email inputs (@example.com), price fields ($), search bars (with button), URL builders, phone numbers with country codes.

* What is ARIA?

ARIA stands for Accessible Rich Internet Applications. It's a set of HTML attributes (like aria-describedby, aria-label, aria-hidden) that help screen readers understand your web page better. Think of ARIA as "extra instructions for assistive technology." When semantic HTML alone isn't enough to convey meaning, ARIA attributes fill in the gaps‚Äîtelling screen readers what elements do, how they relate to each other, and what state they're in. Note: This guide teaches zTheme CSS styling; ARIA is separate from CSS but essential for accessible interfaces.

Basic Input Groups

Wrap an .zInput with .zInputGroup and add .zInputGroup-text for labels or symbols.

<label for="website">Website URL</label>
<div class="zInputGroup">
  <span class="zInputGroup-text" id="url-protocol">https://</span>
  <input type="text" class="zInput" id="website" aria-describedby="url-protocol">

<label for="price">Product Price</label>
<div class="zInputGroup">
  <span class="zInputGroup-text" id="price-currency">$</span>
  <input type="text" class="zInput" id="price" aria-describedby="price-currency">
</div>
Key Accessibility Pattern:
  1. <label> outside the group (linked via for/id)
  2. id on the .zInputGroup-text element
  3. aria-describedby on the input (references the text's id)

Both Sides

Add text or symbols on both sides of an input:

$ .00 
<div class="zInputGroup">
  <span class="zInputGroup-text">$</span>
  <input type="text" class="zInput" placeholder="100">
  <span class="zInputGroup-text">.00</span>
</div>

Multiple Inputs

Combine multiple inputs with shared context:

Name 
<div class="zInputGroup">
  <span class="zInputGroup-text">Name</span>
  <input type="text" class="zInput" placeholder="First" aria-label="First name">
  <input type="text" class="zInput" placeholder="Last" aria-label="Last name">
</div>
Accessibility: Add individual aria-label attributes to each input when using multiple inputs, since they don't have visible labels.

<div class="zInputGroup">
  <input type="text" class="zInput" id="search" placeholder="Enter product name...">
  <button class="zBtn zBtn-info" type="button">Search</button>
</div>

<label for="newsletter">Subscribe to Newsletter</label>
<div class="zInputGroup">
  <input type="email" class="zInput" id="newsletter" placeholder="your@email.com">

Best Practices

Always Use Labels

Place <label> elements outside the input group for screen readers:

<!-- Good: Label outside -->
<label for="email">Email</label>
<div class="zInputGroup">
  <input type="text" class="zInput" id="email" aria-describedby="domain">
  <span class="zInputGroup-text" id="domain">@example.com</span>
</div>

<!-- Bad: No label, no aria-describedby -->
<div class="zInputGroup">
  <input type="text" class="zInput">
  <span class="zInputGroup-text">@example.com</span>
</div>

Connect Text with aria-describedby

Give the .zInputGroup-text an id and reference it in the input's aria-describedby:

<!-- Screen reader will announce: "Price, edit text, described by currency, dollar sign" -->
<div class="zInputGroup">
  <span class="zInputGroup-text" id="currency">$</span>
  <input type="text" class="zInput" aria-describedby="currency">
</div>

Use Buttons for Actions, Spans for Text

If it's clickable, use <button>. If it's just context, use <span class="zInputGroup-text">:

<!-- Good: Button for action -->
<div class="zInputGroup">
  <input type="text" class="zInput">
  <button class="zBtn zBtn-info" type="submit">Search</button>
</div>

<!-- Good: Span for non-interactive text -->
<div class="zInputGroup">
  <input type="text" class="zInput">
  <span class="zInputGroup-text">@example.com</span>
</div>

Use aria-label for Multiple Inputs

When grouping multiple inputs, add individual aria-label attributes:

<div class="zInputGroup">
  <input type="text" class="zInput" aria-label="First name">
  <input type="text" class="zInput" aria-label="Last name">
</div>

Keep Text Concise

Use short, clear text in .zInputGroup-text - symbols often work better than words:

  • ‚ùå "Username prefix:", "Currency symbol:"
