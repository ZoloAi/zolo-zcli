Form Control

Style text inputs, textareas, and form elements with consistent, accessible design

Form Control Elements

Form controls are semantic HTML elements for collecting user input: <input>, <textarea>, and <select>. These elements are crucial for user interaction and should always be paired with <label> elements for accessibility.

Why form controls matter:

  • Accessibility - Screen readers announce labels and input types
  • Mobile UX - Correct input types show appropriate keyboards (email, number, etc.)
  • Validation - Built-in browser validation for email, URL, and other types
  • Usability - Labels make forms easier to understand and use

Basic Semantic Form Controls

Here are form controls using just semantic HTML (no styling classes). The reboot provides minimal default styling:



<label for="basicEmail">Email address</label>
<input type="email" id="basicEmail" placeholder="name@example.com">

<label for="basicTextarea">Message</label>
<textarea id="basicTextarea" rows="3"></textarea>

Accessibility requirement: Every form control must have an associated label. The for attribute on the label must match the id on the input. This connection helps screen readers announce what each field is for.

Enhanced Styling with .zInput

zTheme provides .zInput for consistent, professional form styling with proper spacing, borders, focus states, and organic rounded corners:

<label for="styledEmail" class="zLabel">Email address</label>
<input type="email" class="zInput" id="styledEmail" placeholder="name@example.com">

<label for="styledTextarea" class="zLabel">Message</label>
<textarea class="zInput" id="styledTextarea" rows="3"></textarea>

What the utility classes add:

  • .zInput - Full width, consistent padding, borders, rounded corners, focus states
  • .zLabel - Proper spacing above input, consistent font weight

Sizing

Use .zInput-lg and .zInput-sm to adjust input height for different contexts:

<!-- Large -->
<input class="zInput zInput-lg" type="text" placeholder="Large input">

<!-- Default -->
<input class="zInput" type="text" placeholder="Default input">

<!-- Small -->
<input class="zInput zInput-sm" type="text" placeholder="Small input">

Input States

Disabled

Add the disabled attribute to prevent user interaction and show a disabled appearance:

<input class="zInput" type="text" placeholder="Disabled input" disabled>

Readonly

Add the readonly attribute to prevent editing while keeping the value visible and selectable:

<input class="zInput" type="text" value="Readonly input..." readonly>

Disabled vs. Readonly:

  • Disabled - Grayed out, not focusable, not submitted with form
  • Readonly - Normal appearance, focusable, text selectable, submitted with form

Readonly Plain Text

When displaying read-only data in a form, use .zInput-plaintext to style it as plain text instead of a form field. This removes borders and background while maintaining proper spacing.

<!-- readonly + .zInput-plaintext = plain text styling -->
<input type="text" readonly class="zInput-plaintext" value="email@example.com">
Use Case: Perfect for displaying user profile data, order confirmations, or any read-only information within a form context.

Inline Plaintext

Combine .zInput-plaintext with .zRow and .zCol-auto for inline layouts.

<form class="zRow zG-3" style="align-items: flex-start;">
  <div class="zCol-auto">
    <input type="text" readonly class="zInput-plaintext" value="email@example.com">
  </div>
  <div class="zCol-auto" style="margin-top: 0.5rem;">
    <button type="submit" class="zBtn zBtn-primary">Confirm</button>
  </div>
</form>

Datalists

Datalists allow you to create a group of <option>s that can be accessed (and autocompleted) from within an <input>. These are similar to <select> elements, but come with more menu styling limitations and differences. While most browsers and operating systems include some support for <datalist> elements, their styling is inconsistent at best.

Learn more about support for datalist elements. 
<label for="exampleDataList" class="zLabel">Datalist example</label>
<input class="zInput" list="datalistOptions" id="exampleDataList" placeholder="Type to search...">
<datalist id="datalistOptions">
  <option value="San Francisco">
  <option value="New York">
  <option value="Seattle">
  <option value="Los Angeles">
  <option value="Chicago">
</datalist>
