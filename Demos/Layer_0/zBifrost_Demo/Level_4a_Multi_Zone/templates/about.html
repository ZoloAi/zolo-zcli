A demonstration of zCLI's Flask-like conventions and declarative architecture.

What is zCLI?

zCLI is a modern Python framework that brings Flask's simplicity to the next level with:

  • Declarative Routing - Define routes in YAML, not Python decorators
  • Flask Conventions - Familiar static/ and templates/ folder structure
  • Built-in WebSockets - Real-time communication via zBifrost bridge
  • Auto-Rendering - zDisplay events automatically render to your UI
  • Zero Boilerplate - Focus on your app, not framework setup

Declarative Routing (No Decorators!)

Instead of Flask's decorator approach:

# Flask (imperative)
@app.route("/")
def home():
    return render_template('home.html')

zCLI uses declarative YAML:

# zCLI (declarative)
routes:
  /:
    type: template
    template: "home.html"
    context:
      title: "Home"
WebSocket Integration

Connect to zBifrost in just 3 lines:

const client = new BifrostClient('ws://127.0.0.1:8765');
client.connect();
window.bifrostClient = client;

Check your browser console (F12) to see the connection status!
